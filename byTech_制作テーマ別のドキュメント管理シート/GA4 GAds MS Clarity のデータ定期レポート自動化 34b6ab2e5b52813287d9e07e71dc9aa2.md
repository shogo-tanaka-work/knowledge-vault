# GA4 / GAds / MS Clarity のデータ定期レポート自動化

カテゴリ: マーケティング
ステータス: 保留

**カテゴリ:** ツールPoC（マーケティング）

**期間:** 2025/09/01 - 2025/09/12

**プロジェクトステータス:** 一部保留（認証基盤構築へシフト）

### 1. 背景と目的

- **開始理由:** 営業サイドから要望のあった「GA4、Google広告、Clarity」の重要指標を、Discordへ毎日/毎週/毎月定期的に自動通知するため。
- **期待される効果:** 各種ダッシュボードを巡回する手間の削減と、異常値の早期発見。

### 2. 取り組み内容

- **実施施策:** 1. 各ツールのAPI利用制限（Quotas）の調査。
    1. Google Ads MCC（マルチクライアントセンター）のアカウント作成とテスト環境構築。
    2. Makeによるデータ取得プロトタイプの作成。
- **使用ツール・技術:** Google Analytics API, Google Ads API, Clarity Data Export API

### 3. 進捗と成果

- **調査結果:** GA4は制限に余裕があるが、Clarityは1日10回という厳しい制限があるため、取得タイミングの最適化が必須。
- **判断:** リモート環境（Make等）でのOAuth認証維持が煩雑なため、ローカルMCP接続、または認証基盤を自前で持つ（Vercel/Cloud Run等）構成へ舵を切った。

### 4. 学びとナレッジ

- **技術的な発見:** Google Ads APIの複雑さが最大。OAuth認証の永続化（リフレッシュトークン管理）が実用化の鍵。

### 5. 課題と対応

- **発生した課題:** 営業上の優先度変更と認証の技術障壁。
- **対応方法:** 無理に全自動化を急がず、まずは「ローカル環境で動くMCP」としてデータを取得し、人間がレポート化する工程から段階的に自動化する。

### 6. 今後の展開

- **次のアクション:** 9月以降に来社タイミングを合わせ、GCC（Google Cloud Console）での認証情報作成と、完全自動化に向けた基盤構築を再開する。
