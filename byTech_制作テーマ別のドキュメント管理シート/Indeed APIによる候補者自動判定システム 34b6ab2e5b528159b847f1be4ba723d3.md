# Indeed APIによる候補者自動判定システム

ステータス: 保留

**カテゴリ:** ツールPoC（採用自動化・HR Tech）

**期間:** 2025/09/27 - 2025/10/10

**プロジェクトステータス:** 保留

### 1. 背景と目的

- **開始理由:** Indeedからの応募者情報を自動取得し、初期選考（スコアリング）をAIで行うことで、人事担当者のスクリーニング工数を削減するため。
- **解決したい課題:** 応募者対応のタイムラグ解消と判定基準の標準化。

### 2. 取り組み内容

- **検証内容:** Indeed Apply APIの仕様調査、およびパートナー申請手順の確認。
- **主要な意思決定:** 個人情報を取り扱うため、PoCはGASで行うが、実運用はVPSやMake等のよりセキュアな環境に切り替える方針とした。

### 4. 学びとナレッジ

- **技術的な要件:** 自前でXMLフィードを作成し、Indeedがアクセス可能な場所に配置する「フィード連携」とAPIの組み合わせが必須である点を特定。

### 5. 課題と対応

- **発生した課題:** 既存の採用管理システム（ATS）との競合。
- **対応方法:** 現在利用中のATSがIndeedエントリー連携に対応しているか確認し、API干渉が起きない設計を検討中。

### 6. 今後の展開

- **次のアクション:** パートナー申請の承認待ち。並行して候補者の「スコアリング基準」をLLM向けに定義する。
