# 請求書発行業務のMCP自動化

ステータス: 完了

**カテゴリ:** ツールPoC（業務自動化・MCP）

**期間:** 2025/07/12 - 2025/07/25

**プロジェクトステータス:** 完了（実用化の目処あり）

### 1. 背景と目的

- **開始理由:** 請求書作成における情報収集の手間を解消するため。MCP（Model Context Protocol）の実用性検証も兼ねる。
- **期待される効果:** 請求情報の自動取得からPDF作成までをワンクリックで完結させ、ヒューマンエラーと工数を削減する。

### 2. 取り組み内容

- **実施施策:** Discord（依頼元）→ Googleスプレッドシート（台帳）→ 請求書出力という一気通貫の自動化フロー構築。
- **使用ツール・技術:** Claude Desktop, Discord MCP, Google Sheets MCP
- **主要な意思決定:** 当初検討した「自作サーバによるPDF発行」は保守性が低いため、馴染みのある「Googleスプレッドシートのテンプレート」を活用する方式へ転換した。

### 3. 進捗と成果

- **達成できたこと:** 各種MCPを連携させ、Discordの情報をスプシに転記し、そのまま請求書を生成するフローを確立。
- **定性的な成果:** 実用レベルの出力が確認され、他社事例としても提示可能なクオリティとなった。

### 4. 学びとナレッジ

- **うまくいったこと（Good）:** 複数のMCPサーバーを組み合わせ、ツールを跨ぐ作業をAI上で完結させた点。
- **改善ポイント（Improve）:** GCPのAPI設定や認証情報の管理など、導入時のセットアップが非エンジニアには難解。

### 5. 課題と対応

- **発生した課題:** 請求書サーバの構築ハードルの高さ。
- **対応方法:** スプレッドシートのテンプレート機能を読み込む方式に変更。これにより、フォーマット変更などの微調整を現場で完結できるようにした。

### 6. 今後の展開

- **次のアクション:** 社内での実運用を開始し、エラー率を確認。
- **横展開の可能性:** 「特定プラットフォームの情報を帳票化する」仕組みは、経理だけでなく総務・人事の定型業務にも応用可能。
