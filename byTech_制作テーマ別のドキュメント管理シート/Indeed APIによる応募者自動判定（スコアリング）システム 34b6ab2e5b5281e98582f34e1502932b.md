# Indeed APIによる応募者自動判定（スコアリング）システム

ステータス: 保留

**カテゴリ:** ツールPoC（HR Tech・採用自動化）

**期間:** 2025/10/11 - 2025/10/24

**プロジェクトステータス:** 調査・申請停滞（督促中）

### 1. 背景と目的

- **開始理由:** Indeedのエントリー情報を自動で取得し、あらかじめ定義した基準で候補者をスコアリング（判定）することで、初期選考を自動化するため。
- **解決したい課題:** 応募者へのレスポンス速度の向上、および選考基準の平準化。

### 2. 取り組み内容

- **実施施策:** Indeed Apply API（エントリー連携）の仕様調査、およびIndeedパートナー申請の実行。
- **主要な意思決定:** 機密情報を扱うため、PoC後はVPSやセキュアなiPaaS環境での運用へ移行する方針を策定。

### 3. 進捗と成果

- **達成できたこと:** API利用可否に関する一次調査の完了、および問い合わせ（督促）の実施。
- **定性的な成果:** Indeed側へ送信するための「XMLフィード」の配置が必要である等、技術的要件をクリアにした。

### 4. 学びとナレッジ

- **技術的な発見:** 実運用にはIndeedデベロッパー契約が必須であり、既存の採用管理システム（ATS）を利用している場合は、そのシステム側のAPI対応状況も併せて確認が必要。

### 5. 課題と対応

- **発生した課題:** パートナー申請後のIndeed側からの返答が一切なく、進捗が止まっている。
- **対応方法:** 継続的な督促を実施しつつ、APIを使わない代替案（手動エクスポートデータのAI解析等）の並行検討。

### 6. 今後の展開

- **次のアクション:** 応募者データの受信サーバー構築、およびLLMを用いた詳細な「採用スコアリング基準」の策定。
