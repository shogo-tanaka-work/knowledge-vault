# X（テクじい）自律運用の高度化と自律的エンゲージメント構築

カテゴリ: b-Crew
ステータス: 進行中

**カテゴリ:** SNS自動化・コミュニティマーケティング

**期間:** 2025/12/20 - 2026/02/04

**プロジェクトステータス:** ロジック修正・基盤移行フェーズ

### 1. 背景と目的

- **開始理由:** 単なる情報発信だけでなく、他ユーザーとの交流や最新トレンドの自動解析を含めた「自律型人格」を構築し、コミュニティの熱量を高めるため。
- **解決したい課題:** 重複した内容の投稿防止、および特定の層に偏らない幅広いユーザーへの接触（インプレッション向上）。

### 2. 取り組み内容

- **実施施策:** 1. 重複リストの割愛ロジックの追加。2. 独自ロジックによるエンゲージメント計算（上位5件抽出）。3. 画像投稿APIの安定化（固定画像の選定含む）。
- **主要な意思決定:** 複雑な条件分岐と外部API連携の負荷を考慮し、n8nからNode.js基盤（Render/Vercel）へ完全移行。

### 3. 進捗と成果

- **達成できたこと:** テキスト＋固定画像の自動投稿フローの安定稼働。Render（無料版）での疎通エラーに対するリトライロジックの実装。
- **定性的な成果:** ターゲットとするユーザー層への接触を増やすためのアルゴリズム改修に着手。

### 4. 学びとナレッジ

- **技術的な発見:** X APIでの「下書き保存」が広告アカウント限定である等、仕様上の制限を把握した上での、ダイレクト投稿に特化した設計の重要性。
- **コスト管理:** fal.aiを活用することで、将来的な画像・動画生成のAPIコストを一元管理できる見込みを立てた。

### 5. 課題と対応

- **発生した課題:** Google Cloudの課金エラーにより、Imagen（画像生成）のAPI利用が一時停止。
- **対応方法:** 現時点では無理に課金突破を急がず、ストック済みの固定画像をランダムに選別して投稿する方式へ切り替え、運用を継続。

### 6. 今後の展開

- **次のアクション:** 多田氏と連携した「テストツイート削除」等のクリーンアップ処理の実装。
- **長期的な改善案:** アンケート機能や3択クイズなど、X特有のフォーマットへの自動対応。
