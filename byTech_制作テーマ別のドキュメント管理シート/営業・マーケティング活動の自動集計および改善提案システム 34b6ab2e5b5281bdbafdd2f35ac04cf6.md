# 営業・マーケティング活動の自動集計および改善提案システム

カテゴリ: 営業
ステータス: 進行中

**カテゴリ:** ツールPoC（業務自動化・CRM/SFA連携）

**期間:** 2025/09/27 - 2025/10/10

**プロジェクトステータス:** 進行中（環境整備および一部実装完了）

### 1. 背景と目的

- **開始理由:** GA4、Google Ads、Clarity等の多岐にわたるマーケティング指標と、Zoomでの商談結果を統合的に管理・分析し、ネクストアクションの策定を高速化するため。
- **期待される効果:** 指標の自動通知による「気付き」の誘発と、商談後の振り返り工数の削減による営業生産性の向上。

### 2. 取り組み内容

- **実施施策:** 1. Zoom Webhookを活用した文字起こし・要約のDiscord通知およびスプレッドシート転記。
    1. GAS（Google Apps Script）によるGA4/Clarityの自動通知環境の構築。
    2. Google Ads APIのOAuth認証リフレッシュトークン処理の実装。
- **使用ツール・技術:** GAS, Vercel Functions, Zoom API, Gemini API, GA4/Clarity API

### 3. 進捗と成果

- **達成できたこと:** * セールス1アカウントでの要約・文字起こしのスプシ自動転記・Discord通知の安定稼働。
    - GA4/Clarityの自動通知成功およびGoogleAds開発者トークンの本番昇格。
    - 担当者コメントの抽出ロジック（ユーザ名トリミング、文字数制限考慮）の実装。

### 4. 学びとナレッジ

- **技術的な発見:** Zoom Webhookはレスポンスが遅いとリトライが走るため、即座に200 OKを返す（ADK）処理が重要。
- **運用の知見:** 10万文字を超える長大な文字起こしデータは、スプレッドシートのセル制限を考慮し、複数セルに分割して格納する設計が必要。

### 5. 課題と対応

- **発生した課題:** Zoomの文字起こし単体のWebhookが取得できない。
- **対応方法:** コンパニオン要約のWebhook受信時にUUIDを保持し、後続の定期実行（スプシ監視）で文字起こしデータを紐付け取得する「非同期連携方式」を採用した。

### 6. 今後の展開

- **次のアクション:** 商談内容に対し、「最高の状態」とのギャップをLLMに分析させる高度な振り返り機能の実装。
- **長期的な改善案:** OAuth認証情報の管理をリモート環境（Vercel/Cloud Run等）で完結させる「完全自動化MCP」の構築。
