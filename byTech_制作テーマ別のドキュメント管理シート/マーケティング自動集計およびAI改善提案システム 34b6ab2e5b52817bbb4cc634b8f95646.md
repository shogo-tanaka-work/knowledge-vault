# マーケティング自動集計およびAI改善提案システム

カテゴリ: マーケティング
ステータス: 完了

**カテゴリ:** ツールPoC（マーケティング自動化・データ基盤）

**期間:** 2025/10/25 - 2025/11/07

**プロジェクトステータス:** 安定稼働・週次提案ロジック実装中

### 1. 背景と目的

- **開始理由:** GA4やClarity、Google Adsの散らばったデータを自動集計し、人間が分析する前にAIが「改善の兆し」や「異常値」を提案する仕組みを完成させるため。
- **期待される効果:** 分析業務のルーチン化解除と、広告ROIの最大化。

### 2. 取り組み内容

- **実施施策:** 1. リフレッシュトークンを用いたOAuth認証の完全自動化。
    1. Python実行環境（Pydantic関連）の不具合解消。
    2. ローカルMCP環境の共有および、リモート環境への移行調査。
- **使用ツール・技術:** GA4 API, Clarity API, Google Ads API, Python (MCP), GAS

### 3. 進捗と成果

- **達成できたこと:** アクセストークンの自動更新による1時間制限の突破と、Clarity/GA4のDiscord通知の安定運用。
- **技術的な発見:** ローカルMCPを介して、プロンプト一つで最新の広告数値を取得し、その場でAIと改善策を議論できる環境を構築した。

### 4. 学びとナレッジ

- **技術的な発見:** ライブラリの依存関係（Pydantic等）が環境間で不整合を起こすと自動化が停止するため、仮想環境の固定が本番運用には必須である。

### 5. 課題と対応

- **発生した課題:** GA4で取得すべきパラメータの選別、およびDiscord通知の視認性。
- **対応方法:** 数値の羅列ではなく、前週比などの「変化率」を重視した通知フォーマットへの修正を検討中。

### 6. 今後の展開

- **次のアクション:** GAQL（Google Ads Query Language）を最適化し、より詳細な広告成果（キーワード別等）の取得を開始。
- **長期的な改善案:** 認証情報を安全に保持しつつ、完全にリモート（GAS/Vercel）で完結する自動化基盤の完成。
