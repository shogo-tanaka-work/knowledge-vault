# LINE Message API × Make による業務自動化テンプレート

ステータス: 未着手

**カテゴリ:** ツールPoC（業務自動化・LINE）

**期間:** 2025/08/28

**プロジェクトステータス:** 完了（汎用パターン確立）

### 1. 背景と目的

- **開始理由:** 外部の受講生（岡野さん）の課題解決支援、およびLINE関連の自動化ニーズが高いPoC事例を拡充するため。
- **期待される効果:** ユーザーがLINE公式アカウントに投稿した内容を、リアルタイムかつ正確にExcelへ蓄積するフローの確立。

### 2. 取り組み内容

- **実施施策:** LINE Message APIとMakeを連携させ、Webhook経由で届くJSONデータを構造化してExcel Onlineへ転記するフローを構築。
- **使用ツール・技術:** LINE Message API, Make (旧Integromat), Microsoft 365 Excel
- **主要な意思決定:** メンテナンス性を高めるため、「Parse JSON」モジュールを使わず、Webhookの「サンプルキャプチャ」機能のみで構造化を完結させる手法を採用した。

### 3. 進捗と成果

- **達成できたこと:** LINEに投稿されたテキストが、Makeを介して数秒でOneDrive上のExcelに追加される一気通貫のフロー構築。
- **定性的な成果:** 「MakeのWebhook設定におけるJSON pass-throughの扱い」など、再利用可能なベストプラクティスを抽出できた。

### 4. 学びとナレッジ

- **技術的な発見:** Makeでは、一度テスト投稿を行って「サンプル取得」をさせないと、下流のモジュールでフィールド（変数）として選択できないという手順の重要性を再確認。
- **改善ポイント:** 複雑なJSONも「紫チップ（変数）」で直感的にマッピングできるため、手入力を避けることがエラー防止の鍵となる。

### 5. 課題と対応

- **発生した課題:** SharePoint上のExcelでは認証問題が発生しやすく、自動化が止まるリスクがある。
- **対応方法:** 安定動作のため、個人用またはBusiness用の「OneDrive」に格納されたExcelファイルを標準とするよう設定。

### 6. 今後の展開

- **次のアクション:** Lステップ等、より高度なLINEマーケティングツールとの連携可能性を検証。
- **横展開の可能性:** 予約受付、アンケート集計、簡易日報システムなど、LINEを入り口とするあらゆるToB/ToC業務に転用する。
