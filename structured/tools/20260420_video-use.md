## 2. ツール/機能の基本情報

### 概要
「生の映像フォルダ → Claude Codeにチャット → `edit/final.mp4`」を実現するオープンソースの動画編集自動化ツール。Claude Codeのスキルとして動作し、プリセットやメニュー操作は不要。

### 提供元
- **会社名**: Browser Use（Y Combinator W25採択スタートアップ）
- **公式サイト**: https://browser-use.com
- **GitHub Organization**: https://github.com/browser-use
- **資金調達**: 2025年3月に $17M シードラウンド（Felicis主導）
- **リポジトリ**: https://github.com/browser-use/video-use（スター1,600+）

### 主要機能
- **フィラーワード除去**: 「えーと」「uh」「umm」、無音、言い直しを自動カット
- **自動カラーグレーディング**: warm cinematic / neutral punch などをセグメント別に適用
- **オーディオフェード**: カット箇所に30msフェード挿入（ポップノイズ防止）
- **字幕焼き込み**: カスタムスタイル対応（デフォルト：2-word大文字チャンク形式）
- **アニメーションオーバーレイ**: Manim / Remotion / PIL を使ったグラフィック並列生成
- **自動検証ループ**: レンダリング後に映像ジャンプ・音声ポップ・字幕埋め込みを自己チェック（最大3回）
- **プロジェクトメモリ**: `project.md`に編集履歴を永続化し、続きから再開可能

### 技術スタック・アーキテクチャ
LLMは映像を直接「見ない」。2層構造で「読む」設計：

1. **第1層（必須）**: ElevenLabs Scribe APIによる音声トランスクリプト（単語レベルタイムスタンプ・話者分離）
2. **第2層（任意）**: 必要時の静的フレーム合成

処理パイプライン：
```
Transcribe（ElevenLabs Scribe）
  → Pack（音声+ビジュアルをLLMが読める形式に圧縮）
  → LLM Reasoning（EDL生成）
  → Render（FFmpeg）
  → Self-Evaluation（最大3回自動修正）
  → edit/final.mp4 出力
```

主要言語：Python（75.2%）、HTML（23.7%）

## 5. 比較・優位性分析

### 優位性
- 自然言語のみで編集指示が完結（プリセット・メニュー操作不要）
- 音声トランスクリプトベースで単語境界の精密カットが可能
- OSS・無料（ElevenLabs APIキーのみ必要）
- Claude Codeのスキルとして即座に組み込める
- 自動検証ループによる品質担保

### 劣位性・懸念点
- ElevenLabs Scribeへの外部API依存（コスト・可用性リスク）
- コミット数7件と極めて初期フェーズ（本番利用には不安定な可能性）
- 独立したAPI/SDKがなくClaude Code以外からの利用は想定外
- ライセンスファイルの明示的な記載が未確認

## 6. リスク評価

### ベンダーロックインリスク
- ElevenLabs Scribeへの強依存（音声トランスクリプト処理）
- Claude Code専用スキルのため他のAIエージェントとの連携は困難

### 技術的リスク
- 初期フェーズ（コミット7件）のため破壊的変更が頻発する可能性
- 動画のフレーム直接解析ではなく音声依存のため、映像のみの情報（テロップ等）は処理不可
