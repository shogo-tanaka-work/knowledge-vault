## 2. ツール/機能の基本情報

### 概要
- OpenAIが提供するAIコーディングエージェント。ChatGPTに組み込まれた形で提供
- 2026年4月16日に「Codex for (almost) everything」として大幅刷新
- 並列クラウド処理・非同期タスク管理が設計思想の中心

### 提供元
- OpenAI（https://openai.com）

### 主要機能
- **Computer Use**: Mac/WindowsアプリをCodex自身が直接操作（Figma・Xcode・Slack・ブラウザ等）
- **アプリ内ブラウザ**: Codexアプリ内でWebページをレンダリングし、直接コメントで指示
- **プラグイン（90以上）**: CircleCI / GitLab Issues / Atlassian Rovo / Neon by Databricks / Render / Microsoft Suite / CodeRabbit / Remotion 等
- **MCP（Model Context Protocol）**: サードパーティツールとの連携
- **サブエージェント並列処理**: 複雑タスクを複数エージェントで分散処理

### 技術スタック・アーキテクチャ
- クラウドネイティブ・非同期タスク処理アーキテクチャ
- Codex CLI: Rust実装、オープンソース（GitHub: openai/codex）
- デフォルトモデル: `codex-mini-latest`（o4-miniのCLI最適化版）

## 4. 検証結果

### 定性的評価

#### 機能面の評価
- Computer Useにより「見ながら操作」できる点が他社との大きな差別化
- プラグイン90以上でエコシステムが急拡大
- アプリ内ブラウザはコードとUIのフィードバックループを短縮

#### 操作性・UI/UX
- 非同期・並列タスク管理UIは長時間タスクに適した設計
- 2026年3月にWindowsも対応、プラットフォーム制約が解消

#### 出力品質
- コード品質の盲目レビューでは「優秀」評価が25%（Claude Codeは67%）
- 速度では240+トークン/秒でClaude Codeの約2.5倍

#### 実用性
- 速度・コスト重視のタスク（Express.jsリファクタ等）では圧倒的優位
- 複雑なアーキテクチャ作業はClaude Codeに劣る評価

## 9. 学びとナレッジ

### 発見したこと
- 2026年4月2日に課金モデルがメッセージ単位→APIトークン単位に変更された
- GPT-5.5搭載により同タスクをより少ないトークンで完了できる

- APIトークン単位課金への移行後はキャッシュ活用（75%割引）が重要
