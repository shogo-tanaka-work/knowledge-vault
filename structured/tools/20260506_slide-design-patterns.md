## 1. 背景と検証目的

### なぜこの検証を行うのか
- 法人向けスライド作成需要が高まっており、AIエージェントにスライドデザインの「型」を持たせたい
- note の人気アカウント「パワポ研」(https://note.com/powerpoint_jp) が IR資料を体系的に分類した良質な情報源を発見した
- これを Claude Code のエージェントスキルに落とし込めば、「比較スライド作って」「事業ポートフォリオ可視化」等の依頼に対し、パワポ研由来の鉄板パターン（左右2カラム／自社列をコーポレートカラー強調 等）を参照した提案ができる

### 解決したい課題
- スライド作成依頼に対し、毎回ゼロから構成を考えると質がブレる
- まじん式PPTX生成スキル等は「形式」は出せるが「テーマ別の鉄則」は持っていない
- パワポ研の知見は note 上に分散しており、エージェントが直接参照しづらい

### 期待される効果・ビジネスインパクト
- 比較・KPIツリー・事業紹介・資金調達ピッチなどテーマ別の鉄板構成を即座に提案可能
- パターン辞書 → 既存生成スキル（majin-slide-pptx / navy-blue-pptx）への自動バトンパスで、構成決定〜PPTX出力までを一気通貫
- 法人向け提案資料の初稿作成を分単位に短縮できる見込み

## 2. ツール/機能の基本情報

### 概要
- パワポ研 note の人気78記事から抽出したスライドパターン辞書を参照する Claude Code スキル
- スキル本体は構成提案のみを担当し、PPTX生成は majin-slide-pptx 等の既存スキルに委譲する設計

### 提供元
- 自作（情報源は https://note.com/powerpoint_jp ）

### 主要機能
- A:グラフ系 / B:図解系 / C:レイアウト系 / D:戦略フレーム系 / E:配色 / F:業界別リファレンス / G:メタ の7カテゴリ・約50パターンを辞書化
- 「依頼ワード逆引き」表で、ユーザーの依頼文から該当パターンへ即座に辿れる
- 各パターンに代表記事の note key を埋め込み、`articles/{key}_*.md` で具体例（IR資料の引用元URL付き）を引ける

### 技術スタック・アーキテクチャ
- データ: `structured/projects/powerpoint_jp_research/`（gitignored、画像186MB込み）
- スキル: `.claude/skills/slide-design-patterns/SKILL.md` + `references/` 配下の相対symlink（4階層上の projects/ を指す）
- 取得経路: note 公開API（`/api/v2/creators/...`、`/api/v1/magazines/.../notes`、`/api/v3/notes/{key}`）

## 9. 学びとナレッジ

### 発見したこと
- note には非公式だが安定した公開APIがあり、creators 単位・magazines 単位・個別記事単位で完全取得可能
- magazine API のエンドポイントは v1（`/api/v1/magazines/{key}/notes`）を使う必要があり、v2/v3 は404で罠
- creator API のフィールドは camelCase（`likeCount`、`publishAt`）、magazine API のフィールドは snake_case（`like_count`）と混在している

### うまくいったこと
- API並列＋curlの軽量取得で90メタ＋78本文＋1631画像を数分で完了
- `structured/projects/` を gitignore運用しておいたことで、186MBの画像を安心して隔離できた
- スキル → プロジェクト の参照を相対symlinkにしたことで「将来移動する前提」要件をクリア

### うまくいかなかったこと
- 最初 symlink を3階層相対で貼ってしまい解決失敗（正しくは references/ から ../../../.. の4階層）
- patterns.md の代表記事key、最初は手動穴埋めで `nXXXXXX` プレースホルダ多数 → 後で grep で実keyに置換した

### Tips・ベストプラクティス
- note API のページング判定は `isLastPage` の真偽値を見る（creator）／マガジンは1ページに全件詰まることもあるので空ページ判定を `data.notes | length == 0` で行う
- 画像URLは body内 HTML から `https://assets.st-note.com/...` を正規表現で抽出可能
- スキル description には「依頼キーワードを過剰なほど列挙」した方が発動率が安定する

### よくあるエラーと対処法
- `jq: parse error: Invalid numeric literal` → APIが404 HTMLを返している。エンドポイントが間違い
- `ModuleNotFoundError: No module named 'pptx'` → 環境のpython3が browser-use-env を指していた。`/usr/local/bin/python3` を明示
