## 2. ツール/機能の基本情報

### 概要

Claude モデルにスクリーンショットを与え、`computer_use` ツール経由でマウス・キーボード操作を実行させる仕組み。2024-10 公開以降、2026-05 時点でも Public Beta 継続中。

### 提供元

Anthropic（API 経由 / Amazon Bedrock / Google Cloud Vertex AI で提供）

### 主要機能

- アクション: `screenshot` / `left_click` / `right_click` / `double_click` / `middle_click` / `left_click_drag` / `type` / `key` / `scroll` / `mouse_move` / `cursor_position`
- 新ツールバージョン `computer_20251124` で `zoom`（領域拡大）追加
- Opus 4.7 は長辺 2,576px の大型画像入力に対応

### 技術スタック・アーキテクチャ

- Messages API + `anthropic-beta: computer-use-2025-11-24` ヘッダー
- 公式デモは Ubuntu + X11 + VNC を Docker コンテナで動作

### 優位性

- 視覚認識精度（Opus 4.7 で XBOW 報告 98.5%）
- OSWorld で Opus 4.5 が 66.26% と高水準
- VM / コンテナでの隔離運用が公式ガイドラインで明確
- 大型画像入力（長辺 2,576px）に対応

### 劣位性・懸念点

- claude.ai 通常 UI で利用できないため、エンドユーザー向け面が存在しない
- Beta 継続中で破壊的変更リスクがある
- 同時実行・タイムアウト等の定量制約が公式に明示されていない
- **デスクトップ操作の対象が実質 Linux に限られる**（詳細は §9）

### OS レイヤーの現実（2026/05/28 追記）

CU 各社が「デスクトップ操作対応」と謳う場合、実態は **Linux カーネル系 OS（Ubuntu / Debian / CentOS 等）の上で動くデスクトップ環境を操作する** という意味になる。Anthropic 公式デモも Ubuntu + Xfce + Firefox 構成。

**Windows / macOS の業務アプリ（弥生会計・勘定奉行・Office デスクトップ版・Adobe・AutoCAD 等）を CU で操作したい場合は別途仕掛けが必要**:

| アクセス方式 | 内容 | ライセンス | セキュリティ運用 | 適性 |
|---|---|---|---|---|
| Azure Windows VM + RDP | クラウド VM を立てて CU から RDP 接続 | VM料金にWindows料金込み（月$30〜150） | NSG/Bastion/MFA 必要、設定重め | 企業導入 |
| Windows 365 Cloud PC | Microsoft 純正のクラウド Windows | $24〜162/月/ユーザー | Entra ID 連携が標準 | 企業導入（最もクリーン） |
| dockur/windows 系 KVM コンテナ | Linux ホストの Docker で Windows VM | BYOL（Pro ¥28,380 / Evaluation 90日無料） | 自己責任、社内利用OK | 個人検証・社内PoC |
| AI 専用デスクトップサンドボックス | E2B Desktop / Browserbase Desktop 等 | 秒課金 | ベンダー側で隔離・監査 | スポット利用 |

**macOS の場合**: AWS EC2 Mac (~$1.083/時、24時間最小コミット) が唯一の現実的選択肢。よほどの理由がない限り避ける。

### Windows/Mac アクセスの実運用上の課題

- RDP 許可のための NSG/SG 設定、Bastion / SSM 経由のゲートウェイ構築、証明書認証・MFA 設定が必要で重い
- CU 側のシークレット管理（資格情報の保持方法）を別途設計する必要
- 監査ログ（どのタスクが何を操作したか）を保持する必要
- ネットワーク分離（VM を VPC 内に配置、インターネット直結を避ける）

「ローカル完結の個人向け CU」より「クラウド前提の組織向け CU」のほうが現実的、というのが現在地。検証ラボでは Windows 環境への接続は実機検証していない（コスト・運用負荷がリターンに見合わない判断）。

## 9. 学びとナレッジ

### 発見したこと

- **CU のデスクトップ操作 = 実質 Linux デスクトップ操作**。「デスクトップ対応」という説明は Ubuntu/Debian 系の GUI 上の操作を指しており、Windows/macOS のネイティブアプリは別途 VM + RDP/VNC 接続のレイヤーが必要
- **CU の出番は「Playwright 代替」ではなく「Playwright が苦手な領域の補完」**。公開Webサイト取得は Playwright が圧勝。CU が光るのは①ログイン後の業務SaaS操作 ②DOM変化が激しい新興ツール ③画像・図表ベースの抽出 ④クロスアプリWF ⑤長尾の一回限りタスク ⑥強Bot対策サイト
- **コスト体感**: 1サイト10ステップ実行で各社 $0.05〜0.40 程度（モデル・キャッシュ有無で大きく変動）。Playwright のほぼ$0と比べると1桁以上高い
- **日本企業の業務アプリは Windows ネイティブが多い**（弥生・勘定奉行・PCA等）。これが CU 単独で完結しない最大の制約。Office デスクトップ版・Adobe・AutoCAD も同様

1. **Phase 1**: Web SaaS 業務（営業のCRM、マーケのGA4、CSのチケット起票）から始める。現状コンテナそのままで動く
2. **Phase 2**: Linux ネイティブアプリ業務（Slack/Notion/Zoom/LibreOffice）を Dockerfile 拡張で追加
3. **Phase 3**: Windows 専用業務が必要なら Windows 365 Cloud PC か Azure Windows VM を追加（コスト・運用負荷が一気に上がるので要件確認必須）

**Windows 接続検討時の注意（実機検証は未実施）**:

- 企業導入: Windows 365 Cloud PC か Azure Windows VM + Bastion 経由が標準。NSG/MFA/監査ログ設計が必要
- macOS は EC2 Mac しか現実的選択肢がなく、$1.083/h × 24時間最小コミットでコスト高。基本避ける
