## 1. 背景と検証目的

### なぜこの検証を行うのか
- YouTube のトレンド動画リサーチを手作業でやると時間がかかる
- YouTube Data API を直接叩くのは非エンジニアには敷居が高い
- ノーコードツール（Make / n8n 等）で組むのも面倒で過剰
- そこで「中間案」として GAS + スプレッドシート + 図形ボタンで実装し、画数（手数）がちょうどいいラインを検証する

### 解決したい課題
- トレンドキーワードに対する人気動画リサーチの自動化
- 非エンジニアでも実行可能な配布形態（ボタンクリックで完結）にすること
- 取得項目（タイトル、動画ID、URL、再生数、公開日等）を一覧で出力すること

### 期待される効果・ビジネスインパクト
- リサーチ時間の大幅短縮
- 非エンジニアメンバーへの展開で属人化解消
- Claude Code に自然言語で条件指定 → コード生成 → GAS に貼って実行、というワークフロー自体が他用途にも横展開可能

## 2. ツール/機能の基本情報

### 概要
- GAS のエディタから「サービス」として YouTube Data API v3 を追加すると、`YouTube.Search.list()` などのメソッドが GAS のネイティブ関数のように呼び出せる
- OAuth スコープは初回実行時に承認ダイアログが出るので、そこで許可するだけ

### 提供元
- Google（GAS / YouTube Data API いずれも Google 提供）

### 主要機能
- キーワード検索による動画一覧取得（タイトル、動画ID、公開日、チャンネルID、サムネイル等）
- 検索結果のソート（再生数降順、公開日順、関連度順など `order` パラメータで指定）
- 公開日範囲フィルタ（`publishedAfter` / `publishedBefore`）
- ページトークンによる複数ページ取得（1リクエスト最大50件 → 200件程度まで連結可）

### 技術スタック・アーキテクチャ
- Google Apps Script（V8 ランタイム）
- スプレッドシート（入力キーワード保管 + 出力先）
- 図形描画ボタン → スクリプト割り当てで GAS 関数を発火
- 拡張サービス: YouTube Data API v3（同方式で Gmail API、Tag Manager API、YouTube Analytics API も追加可能）

## 3. 検証方法

### 検証環境
- **使用アカウント**: 個人 Google アカウント
- **プラン/エディション**: 無料枠（Quota: 1日 10,000 ユニット程度）
- **検証環境**: 個人スプレッドシート上で実施

### 検証シナリオ
1. 新規スプレッドシートを開き、拡張機能 → Apps Script でエディタを起動
2. エディタの「サービス」から YouTube Data API v3 を追加
3. キーワードを引数に取る `searchYouTubeKeyword()` を実装、10件取得して Logger 出力
4. 初回実行で OAuth 承認 → 結果確認
5. Claude Code に「再生数降順 / トップ50 / 公開日・URL含む / シートに書き出し」を自然言語で依頼してコード生成
6. 生成コードを GAS に貼り付け → 「ポケモンカード」で実行 → スプレッドシートに書き出される結果を確認
7. 「人気順」「直近3ヶ月」など条件別に関数を分け、シート別に書き込む構成へ拡張
8. スプレッドシート A1 以降にキーワードを並べ、図形ボタンに `RunFromSheet` 関数を割り当てて非エンジニアでも実行できる UI に

### 検証データ・サンプル
- 検索キーワード: 「ポケモンカード」「サッカー」「バスケ」「野球」など
- 取得項目: タイトル / 動画ID / 公開日 / チャンネルID / 再生数 / サムネイル / URL（コード側で組み立て）

### 前提条件・制約事項
- YouTube Data API の Quota（1日あたりのユニット消費量）に注意。`search.list` は1リクエスト100ユニット消費するため、ループで投げると上限に到達しやすい
- 1回の `search.list` で取得できるのは最大50件、それ以上はページトークン連結が必要
- 動画URL は API レスポンスに含まれない → `https://www.youtube.com/watch?v=` + 動画ID で文字列結合して生成

## 9. 学びとナレッジ

### 発見したこと
- GAS の拡張サービス方式なら、APIキー管理が不要で OAuth 一発で済む
- YouTube Data API は動画 URL を返さない → コード側で `https://www.youtube.com/watch?v=` + ID で生成
- Gemini は API 経由で取得した URL を「生成できません」と返す挙動がある（ChatGPT は普通に返す）。Google サービスなのに微妙
- Tag Manager API も GAS から叩ける可能性あり、YouTube に限らず Google タグマネージャーそのものを操作できそう（要追加検証）

### うまくいったこと
- Claude Code に条件を箇条書きで投げる → 生成コードを GAS に貼って実行、で一発で動いた
- 図形ボタン + スクリプト割り当てで非エンジニア配布の形が即作れた

### うまくいかなかったこと
- 一度 500番エラー（サーバー側 `Error, please try again later.`）で実行失敗 → リトライで解消
- シナリオ別関数で書き込み先シートのヘッダー行が空だとエラー
- 複数キーワード処理で一部キーワードの結果が取得できないことがあった

### Tips・ベストプラクティス
- 完了通知の `Browser.msgBox()` は非エンジニアには却って分かりづらい → コメントアウト（行頭に `// ` を付与）するかそもそも実装しない
- 一度に投げるキーワードは Quota の都合で5件程度に制限
- シナリオ別に関数を分け、シートも分けるとあとで条件追加しやすい
- Claude にコード生成を依頼するとき、参考サイト URL（GAS YouTube Data API のリファレンス等）も併せて渡すと精度が上がる

### よくあるエラーと対処法
- `Error, please try again later.`（500系）→ サーバー側問題。少し待って再実行
- `searchYouTubeKeyword is not defined` 系 → 関数名のタイポ、保存忘れ
- 書き込み先シートが存在しない / ヘッダー不一致 → 事前にヘッダー行をコピーしておく
