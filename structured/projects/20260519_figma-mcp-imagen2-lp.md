## 1. 背景と目的
* Imagen2（ChatGPT画像生成）→ Figma でデザインおこし → Figma MCP 経由で Claude Code がコード化、という「画像生成からサイト作成までAI完結」ワークフローが実用に足るか検証する。
* 参考動画のフローを自分の手で再現し、どこまで使えるか・どこに価値が出るかの初感を掴むことが目的。

## 2. 取り組み内容

### 実施した施策・活動
* パワーリフティングアプリ LP のヘッダーメニューとファーストビュー（見た目部分中心）をざっくり作成した。
* 元素材の背景・ボタン表示などは作り込まず、背景を投下する程度のラフな構築にとどめた（細部はこだわらない方針）。
* Figma MCP でデザインデータ（メタデータ・スクリーンショット・画像アセット）を取得し、Claude Code が React + Vite + TypeScript で実装した。
* 成果物は `16_検証ラボ/lab-figma-mcp-imagen2-lp` に配置（Header / Hero / StatsBar、生成画像6点を `src/assets` に格納）。

### 使用したツール・技術
* Imagen2 / ChatGPT（ファーストビュー等の画像生成）
* Figma（生成画像を取り込みデザインおこし）+ Figma リモートMCP（mcp.figma.com）
* Claude Code（Figma MCP からデザイン取得 → React/HTML/CSS 実装）
* React 19 + Vite 6 + TypeScript

### 主要な意思決定とその理由
* 生成画像系（背景・見出し・ダッシュボードUI・ロゴ・特徴アイコン・統計バー）は画像アセットとして忠実配置、ナビ・ボタン・CTAは実テキスト/実ボタンで再構築した。参考動画の「stylized部分は画像、テキスト/インタラクションは再構築」の思想に準拠したため。
* 今回はエージェントスキルなしで実施した（素の状態での実力を見るため）。

## 4. 学びとナレッジ

### うまくいったこと（Good）
* ファーストビュー画像をざっくり入れ込み、画像生成して Claude Code に持ち込み、LP デザインを Figma 上にサクッと作る流れは有効だと感じた。
* 企画初期段階で認識のズレ（齟齬）を潰せる点に価値がある。動くものベースで合意形成できる。
* デザイナー → エンジニアへの「ファーストビューでこんなの作りたい、どうできそう？」のラフ受け渡し用途に向いている可能性。
* 非エンジニア / デザイナー / ディレクターが、顧客要望に対し「こんなのどうですか」と渡す用途にも向いている可能性。
* Figma の作り方・使い方も含めた単純作業は AI に代替され得ると初感で実感。デザイン未経験者でも少し学べば一定品質のものを作れる可能性を感じた。

### うまくいかなかったこと（Bad）
* 出来上がったコードはほぼベタ書きで、コンポーネント分割が甘い（エンジニア目線では要改善）。
* レスポンシブデザインに全く対応しきれていない。チャート/画面縮小時に強く感じた。
* 素材の作り込み不足により違和感のある箇所が残る。

### 改善ポイント（Improve）
* PC/スマホなど各サイズのバージョンを作り分けて読み込めるようにする必要がある。
* 画面縮小時にメニューが見切れる場合、ハンバーガーメニューで開閉できるようにする。
* わかりやすい定型部分は、最初に Figma でデザインを作るより、Claude Code にプロンプト指示して作らせた方が早い場面がある。
* 「良いデザインとは何か」を言語化し、AI に伝えられる力（デザイン原則の理解）を磨くことが重要。

### 技術的な発見・Tips
* 最初からデザインを完成させるより、定型・わかりやすい部分は Claude Code にプロンプトで作らせる方が速いケースがある。
* ファーストビューは「ラフ画像 → 生成 → Claude Code 持ち込み → Figma にデザイン投入」の順が機能する。
* Figma 変数が未定義（全て生成ラスター画像）でも、get_screenshot のネイティブ解像度PNGをアセットとして使えば実装可能だった。

## 7. 今後の展開

### 次のアクション
* レスポンシブ設計（PC/スマホの出し分け、ハンバーガーメニュー）を設計初期から組み込む。
* レスポンシブ対応を支援するエージェントスキルを整備する。
* 【次に試すなら】Claude Code を使わず Codex ベースで実装するワークフローを検証する。
  * 現状フロー: ChatGPTブラウザ版で画像生成 → Figma にインポートしデザイン調整 → Claude Code に MCP接続して書き出し。
  * 検証案フロー: 画像生成を Codex で実施 → 可能なら Codex から Figma へ画像を逆輸入 → デザインを手動で整える → 再び Codex に戻してコード実装まで行う。
  * 微調整は Codex デスクトップアプリ版（プレビュー機能が実装された）でプレビューを見ながら、パディング/マージン等の見た目調整を指示するフロー。
  * 想定構成: Codex の CLI版とデスクトップ版を併用。あるいは Codex + VS Code の組み合わせでも可。
  * 検証ポイント: Codex から Figma への画像逆輸入が可能か、プレビュー連動の見た目調整がどこまで実用的か、Claude Code 非依存で同等品質に到達できるか。
  * 実際に Codex を触ってみての所感（使用感・精度・手戻り量）を記録する。
  * このワークフロー（特に Codex デスクトップのプレビュー連動調整）によって、今回 Bad に挙げたレスポンシブ問題がどこまで解決できるかを重点的に確認する。

### 横展開の可能性
* デザイナー/ディレクター/非エンジニアの「ラフ提案・認識合わせ」ツールとしての活用。
* 顧客要望に対する初期すり合わせ（動く叩き台の即時提示）。

### 長期的な改善案
* 「良いデザインの言語化」スキルの育成（デザイン原則を理解しAIに指示できる人材像）。
* コンポーネント分割など実装品質を担保するための定型プロンプト/スキル化。
