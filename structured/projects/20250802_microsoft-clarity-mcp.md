## 1. 背景と目的
* 開始理由: バイテック公式HPのエンゲージメント情報を、ダッシュボードにアクセスせずMCP経由で取得するため。
* 期待される効果: クラリティへのログイン工数を削減し、チャットUI上で迅速なサイト改善の意思決定を行う。

---

## 2. 取り組み内容

### 実施した施策・活動
* Clarity MCPサーバーを構築し、Claude Desktopからサイト訪問者の挙動データを取得。

### 使用したツール・技術
* Claude Desktop
* Clarity MCP server

### 主要な意思決定とその理由
* 先のGA4検証と同様の操作感を目指し、主要なヒートマップデータやセッション情報の取得を検証。
