# ideas

## 発信に使えそうな切り口

- 筋トレ記録アプリを自作する理由
- AI 時代の個人開発アプリは UI だけでなく API / MCP まで考える
- サボっても戻ってこられる UX
- ローカルファーストな筋トレ記録
- 日本語圏向けの筋トレ分析アプリの余地
- 自分用 MVP から外部公開へ広げる順番
- 無料公開を前提にした収益化候補の整理
- 要件定義で「今やらないこと」を決める意味

## 技術記事にできそうな切り口

- Expo + SQLite でオフラインファーストな記録アプリを作る
- 筋トレログのデータモデル設計
- Web ダッシュボード前提のモバイル DB 設計
- AI 分析に渡すべきデータと渡さないべきデータ
- Codex と要件定義するときの入力ファイル設計
- AI / MCP を後付けしやすいモバイルログ設計
- 個人開発で AI 利用コストと作業ログをどう残すか

## ビジネス向けに使えそうな切り口

- 記録アプリではなく習慣化 OS として価値を定義する
- 筋トレ継続の課題を、通知ではなく意味づけで解く
- サブスク以外の収益化候補を初期から棚卸しする
- 最初のユーザーを自分に絞ることで、検証速度を上げる

## 制作過程で記録したいもの

- 読んだファイル
- 編集したファイル
- Codex への指示
- Codex 側の判断観点
- 実行したコマンド
- 発生したエラー
- 手動で確認できるトークン量や料金
