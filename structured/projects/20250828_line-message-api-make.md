## 1. 背景と目的
* 開始理由: 外部の受講生（岡野さん）の課題解決支援、およびLINE関連の自動化ニーズが高いPoC事例を拡充するため。
* 期待される効果: ユーザーがLINE公式アカウントに投稿した内容を、リアルタイムかつ正確にExcelへ蓄積するフローの確立。

---

## 2. 取り組み内容

### 実施した施策・活動
* LINE Message APIとMakeを連携させ、Webhook経由で届くJSONデータを構造化してExcel Onlineへ転記するフローを構築。

### 使用したツール・技術
* LINE Message API
* Make (旧Integromat)
* Microsoft 365 Excel

### 主要な意思決定とその理由
* メンテナンス性を高めるため、「Parse JSON」モジュールを使わず、Webhookの「サンプルキャプチャ」機能のみで構造化を完結させる手法を採用した。

### 改善ポイント（Improve）
* 複雑なJSONも「紫チップ（変数）」で直感的にマッピングできるため、手入力を避けることがエラー防止の鍵となる。

### 技術的な発見・Tips
* Makeでは、一度テスト投稿を行って「サンプル取得」をさせないと、下流のモジュールでフィールド（変数）として選択できないという手順の重要性を再確認。

## 5. 課題と対応

### 発生した課題
* SharePoint上のExcelでは認証問題が発生しやすく、自動化が止まるリスクがある。

### 対応方法
* 安定動作のため、個人用またはBusiness用の「OneDrive」に格納されたExcelファイルを標準とするよう設定。
