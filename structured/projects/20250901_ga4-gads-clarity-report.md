## 1. 背景と目的
* 開始理由: 営業サイドから要望のあった「GA4、Google広告、Clarity」の重要指標を、Discordへ毎日/毎週/毎月定期的に自動通知するため。
* 期待される効果: 各種ダッシュボードを巡回する手間の削減と、異常値の早期発見。

---

## 2. 取り組み内容

### 実施した施策・活動
* 各ツールのAPI利用制限（Quotas）の調査。
* Google Ads MCC（マルチクライアントセンター）のアカウント作成とテスト環境構築。
* Makeによるデータ取得プロトタイプの作成。

### 使用したツール・技術
* Google Analytics API
* Google Ads API
* Clarity Data Export API

### 主要な意思決定とその理由
* リモート環境（Make等）でのOAuth認証維持が煩雑なため、ローカルMCP接続、または認証基盤を自前で持つ（Vercel/Cloud Run等）構成へ舵を切った。
