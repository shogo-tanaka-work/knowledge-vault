# note 個人発信戦略プラン（2026年5月〜）

## 背景

- 「継続的なストック収益 開発者」のGoogle検索でnote記事が検索上位を独占している事実を観測。発信開始間もない記事も上位に入るレベル。
- 2025年1月のGoogle×note資本業務提携（Google約5億円出資・株式約6%取得）以降、note.com全体のオーガニック流入が月間860万→1,900万PVへ倍増（Ahrefsドメインパワー91）。
- 個人ブログでは検索上位に3〜6ヶ月かかるが、noteなら1〜7日で暫定順位形成、1〜3ヶ月で安定する構造優位がある。
- 2026年3月のGoogleコアアップデートで「薄いコンテンツの排除」が強化され、E-E-A-T観点で一次情報・体験記事の価値がさらに上昇。AI Overview普及で「読んで価値がある記事」しかクリックされない構造に変化。
- 既存資産: `structured/articles/` に20本以上のテーマ記事、`structured/tools/` に60本超の検証ログ。素材は十分だが、note向けに最適化された発信フローが未整備。

## ゴール

1. 検索流入で資産化（ストック型流入の積み上げ）
2. 信頼形成・専門性証明（B2Bリード・MENTA・AI業務自動化診断への導線）

両立するハイブリッド戦略で進める。

## 商品ラインナップ（L3 有料）

2段ロケット構成。メンバーシップはフォロワー1,000人超え到達まで凍結。

| 商品 | 開始タイミング | 価格目安 |
|---|---|---|
| 単発有料記事 | フォロワー300人到達後 | 500〜980円 |
| 5本パック（マガジン or 単記事束ね） | 単発が3本ヒット後 | 単発合計の **30%オフ**（例: 500円×5=2,500円 → 1,750円） |
| ~~月額メンバーシップ~~ | フォロワー1,000人超えで再検討 | (保留) |

**判定ルール**: 単発有料記事は「テンプレ・チェックリスト・固有失敗事例」のいずれかを必ず含む。パック化対象は同一テーマで5本揃ったタイミング（例: ハーネスエンジニアリング5部作）。

## shogoworks 連携座組

**現状**: 保留。フォルダ構造の根本見直し（ソロプレナーAI組織設計＋マーケ目線追加）を別途実施するため、本戦略では決定しない。

**当面の運用**: shogoworks `docs/research/daily-ai-updates/` の日次mdを手動コピペでnote向けに軽整形して投稿（Phase 0）。連携自動化はフォルダ構造見直し後に再設計。

## 公開可否チェックリスト

`outputs/note/disclosure-checklist.md` に `structured/tools/` 60本超の目視判定リストを配置。OK / 要編集 / NG の3値ラベル+1行コメントで管理し、記事化時の元素材選定に使う。

## ペルソナ・ポジショニング

### 一人称の固定

> 現場でAI業務自動化システムを作りながら、100名以上にAI開発を教えてきたフリーランスエンジニア。

### 読者プロファイル

- 主読者: AI活用に踏み出したい非エンジニア事業責任者・PM・営業
- 副読者: Claude Code等を業務に使い始めた他のエンジニア・受託開発者
- 共通点: 「ツール紹介で終わる記事」では満足せず、「実装した結果どうだったか」「失敗とリカバリ」を求めている

### 他noteとの差別化軸

- 横峯樹氏（アソビューCPO）型 → 実装事例の網羅で勝負
- AI FREAK氏型 → 体験談+SEO量産で勝負
- 堀内亮平氏型 → 非エンジニア向け平易解説で勝負
- **shogoworks型 = 「教える×作る」のハイブリッド**:
  - 検証ログを持っている強み（`structured/tools/` 60本超）
  - 100名指導の中で観測した「詰まりポイント」を組み込める
  - → **「自分で試した実装ログ + 受講生がハマった所 + 経営者目線の意思決定基準」を1記事に同居させる**のが他にないポジション

## キーワード戦略（ハイブリッド運用）

### A. ニッチ深掘り型（資産記事 / 月2本）

公式: `ツール名 × 職種・業務 × 悩み` の3語複合

| 例キーワード | 月間検索数目安 | 競合状況 |
|---|---|---|
| Claude Code 非エンジニア 自動化 | 100-300 | 薄い |
| n8n GAS Make 使い分け | 200-500 | 中 |
| AGENTS.md CLAUDE.md 違い | 50-150 | ほぼ無い |
| AI業務自動化 診断 進め方 | 100-300 | 薄い |
| ハーネスエンジニアリング 始め方 | 30-100 | ほぼ無い |

月間検索数100〜500で十分。1記事で月500〜2,000PV、年換算でストック型流入が積み上がる。

### B. トレンド瞬発型（月2-3本）

- 新ツールリリース・公式アップデート発表当日〜翌日に投稿
- 例: Claude新モデル、Codex機能追加、新MCPサーバー発表時
- 短期爆発を狙い、その後ニッチキーワードを足してリライトで資産化

### C. キーワード探索フロー

1. ラッコキーワード（無料）でサジェスト一括取得
2. Googleトレンドで季節変動・急上昇確認
3. 実検索で上位10記事のタイトル・H2を抽出 → 抜けている観点を狙う
4. Google Search Consoleで公開後「インプレッション多×CTR低」キーワードをリライト対象化

## 1記事の品質基準

- 文字数: 3,000〜5,000字（B2B・技術系の上位記事平均）
- タイトル: 30〜35字、キーワード先頭配置
- 見出し: H2を3〜5個、H3を各300〜500字
- 必須3要素:
  1. 自分が試した一次情報
  2. 受講生・相談者がハマった具体パターン
  3. 経営者・PMが意思決定する判断基準
- 画像: 1〜3枚（読了率維持目的）
- CTA: 末尾に MENTA / AI業務自動化診断 / 関連noteへの導線（押し売りせず文脈接続）

## 運用サイクル（週次）

```
月曜: 1週間のキーワード候補3-5本選定（GSC + ラッコ + トレンド）
火-木: 資産記事1本ドラフト（既存structured/からknowledge-transformスキルで変換）
金: 公開
土: トレンド記事を瞬発で1本（必要時）
日曜: 公開後1週間経過記事の順位確認 → リライト判断
```

### 月次目標

- 資産記事 2本 + トレンド記事 2-3本 = 月4-5本
- 公開後1週間で10位以下のものはタイトル・H2・冒頭300字をリライト
- 月末に上位10位以内獲得記事を集計してパターン抽出

## 検証マイルストーン

### Phase 1: 起動確認（2026-06-07時点 / 1ヶ月後）

- 資産記事2本以上を公開できているか
- 1本でもGoogle検索で30位以内に入っているか（GSCで確認）
- 新規note経由のMENTA/問い合わせ流入が観測されたか

### Phase 2: パターン化（2026-08-07時点 / 3ヶ月後）

- 月間note PV合計 5,000以上
- 検索上位10位以内の記事 3本以上
- 上位記事のパターン分析 → 次の四半期戦略にフィードバック

### Phase 3: 収益化検証（2026-11-07時点 / 6ヶ月後）

- 有料記事1本リリース
- メンバーシップ立ち上げ判断
- AI学習対価還元プログラム（2025年8月〜）の収益確認
