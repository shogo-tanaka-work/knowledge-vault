# 個人事業主アプリ開発 受発注フロー ガイド

> 作成日: 2026-05-12  
> 対象: 個人事業主（インボイス登録済）としてアプリ開発を請負で受注する場合の事務フロー全般  
> スコープ: 見積もり後の正式発注 → 受領 → 実装 → 納品 → 検収 → 請求 → 入金 までの一連の事務手続き  
> 補足: 見積もり算定ロジックは別途 knowledge-vault に「AI活用開発の見積もり考え方ガイド」あり（重複回避のため本書では算定式は扱わない）

---

## 1. 受発注フロー全体図

```
[1] 引合い・要件ヒアリング
       │
       ▼
[2] 見積書 発行          ← 受注者（自分）
       │
       ▼
[3] 発注書 受領           ← 発注者 → 自分（保管）
       │
       ▼
[4] 請負契約書（業務委託契約書）締結  ← 双方押印 or 電子契約
       │   ※ NDA があれば先に締結
       ▼
[5] 着手・実装
       │   ※ 仕様変更発生時は変更覚書を都度締結
       ▼
[6] 納品書 発行 ＋ 成果物引渡し  ← 受注者（自分）
       │
       ▼
[7] 検収書 受領           ← 発注者 → 自分（保管）
       │
       ▼
[8] 適格請求書（インボイス）発行 ← 受注者（自分）
       │
       ▼
[9] 入金確認 → 帳簿記帳
```

### 各書類の発行主体と保存期間まとめ

| # | 書類 | 発行 | 受領（自分側保存） | 法定保存期間 |
|---|------|------|------|-------------|
| 2 | 見積書 | 自分 | （控え） | 5年（消費税課税事業者は7年） |
| 3 | 発注書 | 発注者 | ✅ | 5〜7年 |
| 4 | 請負契約書 | 双方 | ✅ | 5〜7年（実務は10年推奨） |
| 6 | 納品書 | 自分 | （控え） | 5〜7年 |
| 7 | 検収書 | 発注者 | ✅ | 5〜7年 |
| 8 | 適格請求書 | 自分 | （控え） | **7年**（消費税法） |

実務上は **全書類7年保存** で統一すれば所得税・消費税双方をカバーできる。

出典: [国税庁 No.5930 帳簿書類等の保存期間](https://www.nta.go.jp/taxes/shiraberu/taxanswer/hojin/5930.htm)

## 2. 書類別 記載要件解説

freee / マネーフォワード等の SaaS が発行する書類は基本的に法令要件を満たしているが、**インボイスの登録番号と税率明示** だけは自分で必ず確認すること。

### 2.1 見積書
- **役割**: 発注前に作業内容・金額・条件を提示。法的義務はないが後のトラブル防止に必須
- **記載**: 発行日、有効期限、品目・作業内容、単価・数量・合計、消費税額、振込先（任意）
- **印紙**: 不要

### 2.2 発注書（注文書）
- **役割**: 発注者が「発注を確定した」意思表示
- **記載**: 発注日、発注番号、品目・作業内容、金額、納期、発注者情報
- **印紙**: 不要
- **注意**: 下請法（取適法）適用取引では発注者の **書面交付義務** に該当。口頭発注は違反

### 2.3 請負契約書（業務委託契約書）
- **役割**: 成果物・納期・報酬・知財帰属・検収条件を合意した法的拘束力ある文書
- **必須項目**: 当事者氏名/住所、業務内容・成果物定義、報酬額と支払条件、納期、検収条件、**知財帰属**、**秘密保持**、契約解除、損害賠償、契約期間
- **印紙税**（紙契約の場合、第2号文書）:
  | 契約金額 | 印紙税 |
  |---|---|
  | 1万円未満 | 非課税 |
  | 〜100万円 | 200円 |
  | 〜200万円 | 400円 |
  | 〜300万円 | 1,000円 |
  | 〜500万円 | 2,000円 |
  | 〜1,000万円 | 10,000円 |
  | 〜5,000万円 | 20,000円 |
  | 記載金額なし | 200円 |
- **電子契約なら印紙不要**（クラウドサイン・DocuSign 等）— 個人開発者は電子契約一択

出典: [国税庁 No.7102 請負に関する契約書](https://www.nta.go.jp/taxes/shiraberu/taxanswer/inshi/7102.htm)

### 2.4 納品書
- **役割**: 成果物を引き渡した証明、検収の起点
- **記載**: 発行日、発注番号、納品物名・数量・単価、合計、納品者情報
- **印紙**: 不要

### 2.5 検収書
- **役割**: 発注者が成果物を確認・承認した証明、**請求書発行の正式トリガー**
- **記載**: 検収日、発注番号、検収品目、合否、検収担当者署名
- **重要**: 「納品したが未払い」のトラブル時、検収書が支払義務発生の決定的証拠になる。**必ず取得**

### 2.6 適格請求書（インボイス）
インボイス登録済み事業者の必須6項目（[国税庁 No.6625](https://www.nta.go.jp/taxes/shiraberu/taxanswer/shohi/6625.htm)）：

1. 発行事業者の氏名・名称および **登録番号（T+13桁）**
2. 取引年月日
3. 取引内容（軽減税率対象品目はその旨）
4. 税率ごとに区分した対価合計額および **適用税率**
5. 税率ごとに区分した **消費税額等**
6. 受領者の氏名・名称

#### 記載フォーマット例（標準税率10%のみのケース）
```
開発費用（10%）        300,000円
  消費税額（10%）       30,000円
  合計                 330,000円
```

#### 端数処理ルール（重要）
- **1請求書につき税率ごとに1回**の端数処理。明細行ごとに丸めて合計するのは違反
- 切捨て・切上げ・四捨五入は任意

#### よくあるミス
1. 登録番号の記載漏れ（無効インボイスとなり相手が仕入税額控除できない）
2. 端数処理を明細行ごとに実施
3. 税率の明示忘れ（金額だけ書いて「10%」を省略）

## 3. インボイス制度 — 2026年5月時点の最新状況

### 3.1 登録番号の表記ルール
- 形式: `T` + 13桁数字（例: `T1234567890123`）
- 個人事業主は法人番号と別体系の固有番号
- 記載位置は書類上部の発行者情報近傍が慣行（法令上の位置指定なし）

### 3.2 経過措置スケジュール（令和8年度税制改正反映）

**売り手側（自分）の2割特例・3割特例**

| 期間 | 内容 |
|---|---|
| 〜2026年9月30日 | 2割特例（売上消費税の20%納付） |
| 2027〜2028年分（個人のみ） | **3割特例**（売上消費税の30%納付）※新設 |
| 2029年以降 | 本則課税 or 簡易課税へ移行 |

**買い手側（発注元）の経過措置 — 免税事業者からの仕入税額控除**

| 期間 | 控除割合 |
|---|---|
| 〜2026年9月30日 | 80% |
| 2026年10月〜2028年9月 | **70%**（改正で延長・緩和） |
| 2028年10月〜2030年9月 | 50% |
| 2030年10月〜2031年9月 | 30% |
| 2031年10月以降 | 0% |

→ 2026年10月以降、発注元の控除率が下がるため値引き交渉圧力が高まる可能性あり。事前に納税シミュレーションを実施しておくこと。

出典: [国税庁 インボイス制度について](https://www.nta.go.jp/taxes/shiraberu/zeimokubetsu/shohi/keigenzeiritsu/invoice_about.htm) / [フリーランス協会ニュース](https://blog.freelance-jp.org/20251217-24616/)

### 3.3 電子帳簿保存法との関係
- 電子データで発行・受領した適格請求書は **電子データのまま保存義務**（印刷紙保存は不可）
- 「真実性確保」と「可視性確保」の両要件を満たすこと
- 個人事業主の現実解: **事務処理規程（国税庁ひな形）＋フォルダ運用**

#### 推奨フォルダ運用
```
受領フォルダ:
  2026/受領/2026-05-15_株式会社XX_330000_発注書.pdf
発行控え:
  2026/発行/2026-05-15_株式会社XX_330000_請求書.pdf
```
- 命名規則: `日付_取引先_金額_書類種別.pdf`
- 日付・取引先・金額で検索可能 → 可視性要件を充足
- 「電子取引データの訂正および削除の防止に関する事務処理規程」を作成・遵守 → 真実性要件を充足（国税庁公開ひな形を流用）

出典: [国税庁 電子帳簿等保存制度特設サイト](https://www.nta.go.jp/law/joho-zeikaishaku/sonota/jirei/tokusetsu/index.htm)

## 4. 下請法・契約上の注意点

### 4.1 下請法（2026年1月「取適法」へ改称・拡充）の適用条件

アプリ・プログラム開発は「情報成果物作成委託」に該当。下請法が適用される組み合わせ：

| 委託元（親事業者）資本金 | 受託者（自分）資本金 |
|---|---|
| 5,000万円超の法人 | 5,000万円以下 or 個人 |
| 1,000万円超〜5,000万円以下の法人 | 1,000万円以下 or 個人 |

**個人事業主は資本金0扱いのため、発注元が資本金1,000万円超法人なら原則保護対象。**

2026年1月の取適法施行で **従業員数基準（プログラム作成委託は従業員300人超）** が追加。

出典: [公正取引委員会 下請法の概要](https://www.jftc.go.jp/shitauke/shitaukegaiyo/gaiyo.html) / [取適法リーフレット](https://www.jftc.go.jp/file/toriteki_leaflet.pdf)

### 4.2 親事業者の4義務
1. **発注書面交付義務**（口頭発注は違反）
2. **支払期日の設定義務**（受領から60日以内）
3. **遅延利息支払い義務**（年率14.6%）
4. **書類保存義務**（2年間）

### 4.3 アプリ開発で頻発する禁止行為
- **買いたたき**: 工数・スキルを無視した一方的固定単価
- **不当な給付内容の変更・やり直し**: 仕様変更を無償でやらせる
- **支払遅延**: 検収を意図的に遅延させるケース含む
- **受領拒否**: 主観的理由での受取拒否
- **不当な経済上の利益提供要請**: 無償の保守・バグ対応強要
- **（2026年新設）協議なしの一方的代金決定**

### 4.4 フリーランス保護新法（2024年11月施行）

下請法が適用されない発注元（資本金1,000万円以下法人・個人事業主）でもカバーされる。

| 項目 | 下請法（取適法） | フリーランス保護新法 |
|---|---|---|
| 適用発注元 | 資本金1,000万円超法人 | **従業員1人以上の全事業者** |
| 適用受託者 | 個人 or 中小法人 | **従業員なしの個人事業主のみ** |

**押さえるべき5ポイント**
1. 取引条件の書面明示義務（業務内容・報酬・支払期日を即時書面化）
2. 報酬60日以内支払い義務
3. 募集情報の的確表示義務
4. ハラスメント防止措置義務
5. 6カ月以上継続契約の中途解除は30日前予告義務

相談窓口: **フリーランス・トラブル110番**（公正取引委員会）

出典: [公正取引委員会 フリーランス法特設サイト2025](https://www.jftc.go.jp/freelancelaw_2025/)

### 4.5 知的財産権の帰属 — 個人開発者の推奨スタンス

著作権法第17条により、契約がなければ著作権は **制作者（受託者＝自分）に原始帰属**。

| パターン | メリット | デメリット |
|---|---|---|
| 完全譲渡（買取り） | 発注者が自由に改変可。対価上乗せ可能 | 同種コードの再利用が制限される |
| **利用許諾（ライセンス）** | 汎用コード再利用可。次案件への展開を守れる | 発注者が改修で第三者を入れにくい |
| 共同保有 | — | 共有者全員同意要（著作権法第65条）。実務上は権利行使が複雑 |

**推奨**: プロジェクト固有部分は譲渡、汎用モジュール（ユーティリティ・自作ライブラリ）はライセンス留保する **分離方式**。共同保有は原則回避。

### 4.6 追加要件発生時の運用

1. 基本契約に「仕様変更は書面合意・別途見積もり」と明記
2. 変更管理票（Change Request）を都度起票
3. 着手前にメール等で「追加費用〇円が発生します」と通知し承諾取得
4. 議事録を毎回送付（異議なしで同意とみなす旨を記載）
5. 確定したら **変更覚書** を締結してから作業着手

### 4.7 支払遅延発生時の対応手順

1. 催促メール（支払期日・未払金額を明記、書面で証跡化）
2. 遅延利息請求通知（年14.6%、下請法/フリーランス法適用取引）
3. **フリーランス・トラブル110番** に相談（無料・匿名可、報復禁止規定あり）
4. 内容証明郵便で催告
5. 支払督促 or 少額訴訟（60万円以下）

### 5.3 推奨

## 6. フェーズ別チェックリスト（要約）

詳細は `assets/freelance-app-dev-workflow/checklists/` 配下参照。

### 着手前（契約締結まで）
- [ ] 見積書発行・有効期限明示
- [ ] 発注書受領（口頭発注NG、書面 or 電子データ）
- [ ] 請負契約書締結（電子契約推奨で印紙ゼロ）
- [ ] NDA 必要なら先行締結
- [ ] 知財帰属条項を分離方式で記述（汎用部分はライセンス留保）
- [ ] 検収条件・変更管理プロセスを契約書に明記

### 納品前
- [ ] 仕様書/契約書との差異がないか自己レビュー
- [ ] 変更覚書ベースの追加要件もすべて反映済みか確認
- [ ] 納品書発行
- [ ] 検収書フォーマットを発注者に渡し検収を促す

### 請求時
- [ ] 検収書受領済み
- [ ] 適格請求書6項目すべて記載（特に **登録番号 T+13桁**）
- [ ] 端数処理は税率ごと1回
- [ ] 電子発行ならPDFをフォルダ運用ルールで保存
- [ ] 支払期日（受領から60日以内）を確認

### 入金後
- [ ] 入金消込
- [ ] 帳簿記帳（freee会計推奨）
- [ ] 関連書類7年保存（受領/発行両方）
