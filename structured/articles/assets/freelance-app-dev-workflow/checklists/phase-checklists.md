# フェーズ別チェックリスト

個人事業主（インボイス登録済）アプリ開発受注のフェーズ別実務チェック。各フェーズ完了時にこのチェックを通すことで漏れを防ぐ。

---

## Phase 0: 引合い・初回ヒアリング

- [ ] 発注元の会社規模（資本金・従業員数）を把握 → 下請法 or フリーランス保護新法どちらの適用かを判定
- [ ] 案件規模・予算感のすり合わせ
- [ ] スコープと非対象範囲を口頭で確認（後で書面化）
- [ ] 機密情報を扱う場合は **NDA を先行締結**

---

## Phase 1: 見積もり提示

- [ ] 見積書発行（PDF or freee等SaaS）
- [ ] 有効期限を明示（推奨: 発行から30日）
- [ ] 内訳を「フェーズ」「作業内容」「工数」「単価」「小計」で記載
- [ ] 消費税を区分表示
- [ ] 振込先・支払条件を記載（任意だが推奨）
- [ ] 見積条件に「本見積もりは仕様確定前のため、要件変更により金額が変動する可能性がある旨」を注記

---

## Phase 2: 正式発注 → 契約締結（着手前）

### 発注書受領
- [ ] 発注書を **書面 or 電子データ** で受領（口頭発注はNG）
- [ ] 発注書記載事項（発注日・品目・金額・納期・発注者情報）を確認
- [ ] 受領した発注書PDFを `2026/受領/YYYY-MM-DD_取引先_金額_発注書.pdf` で保存

### 請負契約書締結
- [ ] `templates/contract-ukeoi.md` ベースでドラフト作成
- [ ] 知財帰属条項を **分離方式**（プロジェクト固有部分=譲渡 / 汎用モジュール=ライセンス留保）で記述
- [ ] 検収条件・期日（推奨10営業日、無通知合格条項あり）を明記
- [ ] 仕様変更プロセス（書面合意＋変更覚書）を明記
- [ ] 損害賠償の上限額を委託料額に限定
- [ ] 中途解約条項（継続契約6ヶ月以上は30日前予告）
- [ ] **電子契約で締結**（クラウドサイン等）→ 印紙税ゼロ
- [ ] 契約書PDFを `2026/契約/YYYY-MM-DD_取引先_業務委託契約書.pdf` で保存

### 着手前の最終確認
- [ ] 業務仕様書（契約書 別紙）が確定し合意済み
- [ ] 環境（GitHub・Supabase・Vercel等）のアクセス権を発注元から受領
- [ ] 連絡チャネル（Slack/Email/Notion）を確定

---

## Phase 3: 実装中（仕様変更が発生したら）

仕様変更が発生したら **着手前に** 以下を全て実施：

- [ ] 変更要件をメール or チャットで明文化
- [ ] 工数・追加費用を見積もり、発注元に通知
- [ ] 「追加費用 ○○ 円が発生します。承諾いただけますか？」と明示確認
- [ ] 発注元の承諾を **書面（メール返信可）** で取得
- [ ] `templates/change-order.md` を使い変更覚書を起草
- [ ] 電子署名で締結後に作業着手

⚠️ 口頭の「これもお願い」「ちょっと変えて」は一切受けない。受けたら必ず書面化プロセスへ。

---

## Phase 4: 納品

- [ ] 仕様書（契約書＋全変更覚書）に対する成果物の照合チェック
- [ ] テスト結果・動作確認エビデンスをドキュメント化
- [ ] 納品物のリポジトリ・URL・認証情報を整理
- [ ] 納品書を発行（SaaSテンプレでOK）
- [ ] **検収書フォーマットを発注元に提供**（自分で雛形を渡すと検収完了が早い）
- [ ] 納品方法（GitHub push / ZIP送付等）を契約書記載の方法で実行
- [ ] 納品書PDFを発行控えとして保存

---

## Phase 5: 検収 → 請求

### 検収書受領
- [ ] 検収書を受領（または契約上の自動検収期日経過を確認）
- [ ] 検収書PDFを `2026/受領/YYYY-MM-DD_取引先_金額_検収書.pdf` で保存
- [ ] 検収不合格の場合、不合格理由が **仕様書範囲内** か精査（仕様外なら仕様変更扱い）

### 適格請求書発行（インボイス6項目チェック）
- [ ] ① 発行事業者の氏名・名称
- [ ] ① **登録番号（T + 13桁）** ← 最頻出のミス
- [ ] ② 取引年月日
- [ ] ③ 取引内容
- [ ] ④ 税率ごとの対価合計＋適用税率（10%）
- [ ] ⑤ 税率ごとの消費税額
- [ ] ⑥ 受領者の氏名・名称
- [ ] **端数処理は1請求書につき税率ごとに1回**（明細行ごとに丸めない）
- [ ] 支払期日を明記（受領日から60日以内）
- [ ] 振込先口座を明記
- [ ] 請求書PDFを `2026/発行/YYYY-MM-DD_取引先_金額_請求書.pdf` で保存

---

## Phase 6: 入金 → 帳簿

- [ ] 支払期日に入金確認
- [ ] **入金が遅延した場合**:
  - [ ] 催促メール（書面で証跡化）
  - [ ] 遅延利息（年14.6%）の請求権を認識
  - [ ] フリーランス・トラブル110番に相談（無料・匿名可）
- [ ] 入金消込
- [ ] 帳簿記帳（freee会計推奨。freee-mcp 経由で自動化可）
- [ ] 関連書類すべて7年保存（受領フォルダ・発行フォルダ・契約フォルダ）

---

- [ ] 2割特例（2026年9月まで）/ 3割特例（2027〜2028年）/ 簡易課税の選択を毎年見直し
- [ ] 経過措置スケジュールに合わせて納税シミュレーション
- [ ] 「電子取引データの訂正および削除の防止に関する事務処理規程」の運用状況を確認
- [ ] 関連書類が7年保存ルールに則って整理されているか棚卸し
