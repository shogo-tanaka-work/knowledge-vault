## 投稿1（気づき・単独で成立）

投稿本文 ↓

毎週月曜にYouTubeのトレンドを眺める地味な作業、Claude CodeとBrowser Use CLIに任せたら景色が変わった。

YouTube Data APIにShorts専用枠はないし、Google トレンド「探す」タブも公式APIでは取れない。結局ブラウザを自動で動かす方向に振り切って、Claude Codeから1コマンドで叩く形にした。手作業30分がゼロになる。

---

## 投稿2（整理／段階提示）

投稿本文 ↓

📌 仕組みの全体像

・Claude Codeのスキルに登録
・Browser Use CLIでChromeを自動操作
・Google トレンドの「探す」タブをYouTube＋期間指定で開く
・12秒待ってDOMから取得、上位50件を整形

肝はURLにパラメータを全部埋め込むこと。date、geo、gprop、hlを変数化しておけば「先週分」「3ヶ月分」が一発で切り替わる。

---

## 投稿3（詰まったところ）

投稿本文 ↓

💡 一番ハマったのは待ち時間

開いてすぐテキストを取ると空が返る。Google トレンドはJS描画が遅くて、短いsleepだと安定しない。結局12秒固定にした。

もう一つはアンチボット。素のChromiumだと弾かれる週があって、実機のChromeプロファイルを指定する方式に切り替えたら安定した。AIに任せる部分と、人間の環境の匂いを残す部分のバランスが効く。

## 投稿4（比較・対比）

投稿本文 ↓

👉 公式APIとブラウザ自動操作の使い分け

▼ YouTube Data API v3
・無料枠あり、安定
・Shorts指標は2026年も未公開
・「探す」タブ系は取れない

▼ Browser Use CLI経由の自動化
・UI経由で公式未公開の指標も取れる
・JS描画待ちとアンチボット対策が要る
・規約遵守は自己責任

定型集計はAPI、一次情報の収集はブラウザ、で住み分けると一気に楽になる。
