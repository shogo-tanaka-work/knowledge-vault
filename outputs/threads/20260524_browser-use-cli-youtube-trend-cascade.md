## 投稿1（気づき・単独で成立）

投稿本文 ↓

毎週月曜にYouTubeのトレンドキーワードを眺める、っていう地味な作業を半年くらい続けてたんだけど、これをClaude CodeとBrowser Use CLIに任せたらだいぶ景色が変わった。

最初はAPIで取れないかと思ったんだよね。でもYouTube Data APIは「Shorts専用エンドポイント」が今も無い。Google トレンドの「探す」タブのデータも公式APIで取れない。

で、結局ブラウザを自動で開いて中身を読み取る方向に振り切った。AIエージェントがブラウザを操作するBrowser Use CLIを噛ませて、Claude Code側からコマンドで指示する形にしてある。手作業30分が1コマンドになった。

## 投稿2（整理／段階提示）

投稿本文 ↓

📌 仕組みの全体像

・Claude Codeのスキルとして登録
・Browser Use CLIでChromeを自動操作
・Google トレンドの「探す」タブをYouTube+期間指定で開く
・12秒待ってDOMからテキストごと取得
・上位50件を整形して出力

設計で意識したのは、検索URLにパラメータを全部埋め込むこと。
date=today%201-m（過去1ヶ月）、geo=JP、gprop=youtube、hl=ja。ここを変数化しておけば「先週分」「3ヶ月分」みたいな切り替えが一発でできる。

スクレイピングというより、ブラウザを目視確認の代わりに動かしてる感覚に近い。

## 投稿3（詰まったところ）

投稿本文 ↓

💡 一番ハマったのは「待ち時間」

最初、ページを開いてすぐテキストを取りに行ったら空っぽが返ってきた。Google トレンドのページって、開いた直後はJS描画が完了してなくて、トレンドリスト部分が空のまま。

短いsleepで何度か試したけど安定しなくて、結局12秒待つことにした。1コマンドの中で済む話なのに、待ち時間の調整に半日溶けた感じ。

あと、普通のChromiumだとアンチボットに弾かれる週があった。実機のChromeプロファイルを指定して開く方式に切り替えたら安定した。AIに任せられる部分と、人間の環境の匂いを残しておく部分のバランスが意外と大事だった。

## 投稿4（比較・対比）

投稿本文 ↓

👉 公式API vs ブラウザ自動操作、自分の選び方

▼ 公式API（YouTube Data API v3）
・無料枠あり、安定
・Shorts専用の指標は2026年も未公開
・「探す」タブ系のトレンドは取れない

▼ Browser Use CLI 経由のブラウザ自動化
・公式に出てない指標もUI経由で取れる
・JS描画待ち・アンチボット対策が要る
・規約遵守は自己責任（個人の情報収集スコープ）

結論、定型集計や数値分析は公式API、ヒトの目で見てた一次情報の収集はブラウザ自動化、っていう住み分けに落ち着いた。両方使い分けると一気に楽になる。
