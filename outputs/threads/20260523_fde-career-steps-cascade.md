# Threads投稿案：FDEの定義とキャリアステップを整理した話

5本連投を想定。新着順アルゴリズムを狙い、短時間に投下する。
各投稿は会話体・400〜500字以内。投稿1は単独でも成立する形にする。

---

## 投稿1（気づきベース）

```
FDE（Forward Deployed Engineer）って結局なんなんだろう、と前から気になってた。OpenAIやAnthropicの求人を見るとよく出てくる肩書きなんだけど、上位エンジニアともAIコンサルとも微妙に違う。

少し時間をとってAIと壁打ちしながら整理してみたら、いちばんしっくりきたのは「AI時代の業務実装責任者」っていう表現だった。PoCで終わらせず、本番導入から業務インパクトまで持ちきる役割。エンジニア・業務コンサル・ソリューションアーキテクト・PMを少しずつ混ぜた総合職、と言われるとイメージしやすい。
```

---

## 投稿2（5段階の整理）

```
FDEに至るキャリアを5段階で整理してみた。

1. AIで小さく作れる（RAG・チャットbot・業務自動化）
2. 業務フローに組み込める（CRM・Slack・社内DB連携）
3. 本番運用を回せる（評価・ログ・LLMOps）
4. コストとリスクを語れる（FinOps・ガバナンス）
5. 現場で成果まで持ちきる（FDE）

最初からFDEを目指す感じではなくて、1〜4を順番に積んだ先に5へ自然に届く構造になってる、と整理してから少し腑に落ちた。
```

---

## 投稿3（日本の現実解釈）

```
日本ではFDEっていう肩書きはまだ定着してない。でも、業務改善×情シス×SI×AI導入コンサルを横断できる人の需要は確実に増えてる印象がある。

肩書きがFDEじゃなくても、中身はほぼFDEっていう形が、いまの日本では現実的だと思ってる。むしろ無理に外資の肩書きを名乗らず、日本の現場に合わせた立ち位置で動いたほうが、刺さるはず。AnthropicやOpenAIの求人を参考にしつつ、日本流の動き方に翻訳していく、という感覚。
```

---

## 投稿4（大企業 vs 小規模）

```
面白いのは、大企業向けFDEと小規模事業者向けFDEで、求められる能力が全然違うこと。

大企業は、CRM・ERPに統合してCTO・CIO・法務を説得する仕事。意思決定者が多く稟議も長い。

小規模事業者は、経営者本人と直接やる仕事。判断基準は、来週使えるか・月3万円以内か・自分でも触れるか。技術の高度さじゃなくて、ツール選定眼が核心になる。スクラッチに走る前に、NoCodeとSaaSで解決できるかを見極められる力。

後者に強い人材、日本ではまだ希少だと感じる。
```
