# 数ヶ月検索に出なかった自社サイトを、Google検索とAI検索の両方に乗せた話

## 起きていたこと

自社サイト（shogoworks.com）を公開して数ヶ月、Google で `site:shogoworks.com` と検索しても1件も出てきませんでした。サイト自体はちゃんと表示されます。なのに検索結果には存在しない。788ページのナレッジを載せているのに、誰にも見つけてもらえない状態でした。

調べていくと、原因は1つではありませんでした。Google検索に出ない問題と、AI検索（ChatGPTやGeminiなど）に出ない問題が、別々に絡んでいたのです。順番に解いていきます。

## 全体像

サイトは Astro で作り、Cloudflare Pages にホスティング、ドメインは Squarespace で管理しています。やったことを図にすると、こうなります。

```
公開しただけ（＝見つからない）
   ↓
① Google Search Console に能動登録
   ↓
② Astro でサイトマップを生成して送信（788ページ検出・成功）
   ↓
③ Cloudflareの「AI向けrobots.txt自動生成」をOFF
   ↓
④ robots.txtを粒度制御（検索OK・AI検索OK・学習はNO）
```

（※Search Consoleで788ページ検出・成功になった画面のスクリーンショットを1枚挿入：要追記）

## いちばんのキモは「robots.txt の二重管理」だった

問題2でしばらく沼にはまりました。サイトの設定ファイル（robots.txt）を正しく書き換えて、AI検索のクローラーを通すようにしても、デプロイするたびに元のブロック設定が復活するのです。

原因は、Cloudflare がこのファイルを二重に管理していたことでした。自分が書いた robots.txt とは別に、Cloudflare の AI向け機能（AI Crawl Control の Managed robots.txt）が、AIクローラー用のブロックを自動で挿入していたのです。robots.txt は上から順に読まれるので、先頭に差し込まれたブロックが、自分が後ろに書いた許可より優先されていました。

解決はシンプルで、Cloudflare の管理画面でこの自動生成機能をOFFにするだけでした。設定したのに効かないときは、どこかで上書きされていないかを疑う。これが今回いちばんの学びです。

## Before / After

| | Before | After |
|---|---|---|
| Google検索 | `site:` で0件、存在しない扱い | サイトマップ788ページ検出・成功 |
| AI検索 | クローラーをブロックしたまま | ClaudeBot・GPTBot・Geminiに引用される設計へ |
| robots.txt | Cloudflareに上書きされ意図と逆 | 自分の設定がそのまま反映 |
| 権利管理 | 全許可か全拒否の二択 | 検索OK・AI検索OK・学習はNOの粒度制御 |

最後の robots.txt は、検索とAI検索の引用は許可しつつ、学習データとしての収集は拒否する形にしました。AIに引用はされたいけれど、勝手に学習素材にはされたくない。そのさじ加減を、設定ファイルの中で宣言できます。

## こんな場面で効くと思います

- サイトを公開したのに検索に出てこなくて、原因が分からず放置しているとき
- これからは Google だけでなく AI検索にも引用されたい、けれど学習利用は線引きしたいとき
- Cloudflare や Astro で作ったサイトの「設定したのに効かない」に詰まっているとき

ナレッジ記事を多く持つサイトほど、AI検索で引用される効果は大きくなります。検索エンジンの入口に加えて、AIの回答経由の流入という新しい経路を、ゼロ円で開けます。

## 教訓

新規ドメインは、公開しただけでは見つかりません。被リンクがゼロのサイトは自然クロールを待っても来ないので、Search Console への能動登録が必須です。そして AI検索時代の robots.txt は、全許可と全拒否の二択ではなく、検索・AI検索・学習を個別に制御する発想が要ります。

設定が効かないときは、自分の設定の正しさを疑う前に、それがどこかで二重管理・上書きされていないかを見にいく。遠回りに見えて、これがいちばんの近道でした。
