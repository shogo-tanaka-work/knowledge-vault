# AIに任せて、自分用のiOSアプリを実機まで作ってみた

## やりたかったこと

筋トレの記録アプリがほしいと前から思っていました。世の中にアプリは山ほどありますが、入力が面倒だったり、後から記録を直せなかったりで、どれも自分の使い方に微妙に合いません。

そこで、AIコーディングツールの Claude Code に任せて、自分専用の記録アプリをゼロから作れるか試しました。ゴールは「App Store に出す」ではなく、まず自分の iPhone に入れて、ジムで実際に使えるところまで届かせることです。

結論を急がずに言うと、企画から実機インストールまで一日で到達できました。ただし、つまずいたのはコードそのものではなく、その周辺でした。ここが個人開発を始める人にいちばん伝えたい部分です。

## 全体の流れ

作ったのは記録に特化したシンプルな構成です。技術スタックは Expo（React Native）と TypeScript、保存はスマホ内のデータベース（SQLite）。Claude Code に要件と画面方針を渡し、実装・ビルド・デバッグの大半を任せました。

```
企画・要件の整理
   ↓
画面設計（記録 → 履歴 → 種目管理）
   ↓
Claude Code が実装（記録ループ・休憩タイマー・履歴集計）
   ↓
シミュレータで動作確認
   ↓
自分のiPhoneへ実機インストール（単体で動く版）
```

（※実機で動いている画面のスクリーンショットを1枚挿入：要追記）

記録を始めて種目を足し、セットごとに重量と回数を入れる。終わったら履歴で振り返る。この中核のループは、ほぼ詰まらずに動くものができました。

## いちばんのキモは「日本語フォルダ名」だった

意外な落とし穴がここです。プロジェクトを日本語名のフォルダに置いていたところ、iPhone 向けのビルドが途中で止まりました。

原因は、iOS のビルドが内部でフォルダのパスを特殊な形式（URL のような文字列）に変換する際、日本語が混ざると処理に失敗することでした。アップル系の開発ツールは、いまも日本語や空白を含むパスに弱いところがあります。

対処はシンプルで、プロジェクト配下のフォルダ名をすべて英語に変えてパスを半角英数字だけにしました。これでビルドはあっさり通りました。コードは1行も悪くなかったのに、置き場所の名前で半日溶けるところでした。これから個人開発を始めるなら、コードを置くフォルダは最初から英語名にしておくと安全です。

## Before / After

| | Before | After |
|---|---|---|
| 記録アプリ | 既製品で入力が面倒、後から直せない | 自分の使い方に合わせて自作 |
| 開発の障壁 | 「アプリ開発は専門職の領域」という思い込み | AIに任せれば、企画者でも実機まで届く |
| つまずき | コードが書けないこと、だと思っていた | 実際は環境構築・署名・端末設定が壁だった |

実機に入れる最後の工程でも、署名の許可や、iPhone側の開発者向け設定の有効化など、細かな関門がいくつもありました。ただ、それらはエラーメッセージを Claude Code に読ませれば、ほぼその場で解決方法までたどり着けます。

## こんな場面に効くと思います

- 受託や社内導入で「まず動くもので見せて」と言われたが、外注に出す前に当たりをつけたいとき
- 既製ツールが自社の業務に合わず、専用の小さなアプリを試作したいとき
- 非エンジニアの企画者が、要件を言葉だけでなく動くプロトタイプで共有したいとき

費用はゼロで、追加投資の判断材料になる「実物」が手元に残ります。外注の見積もりを取る前に、作りたいものの解像度を一段上げられるのは大きいはずです。

## これからやること

正直に書くと、まだ完成ではありません。前回のトレーニング実績を入力画面で見られるようにする部分や、種目ごとの推移グラフは、これから作り込みます。それでも「自分の iPhone で、自分の作ったアプリを使う」ところまでは確かに届きました。

AIに任せられる範囲が広がったいま、開発の入り口に立つハードルはかなり下がっています。詰まるのはコードよりも環境です。逆に言えば、そこさえ越えれば、企画する人がそのまま作る人になれます。
