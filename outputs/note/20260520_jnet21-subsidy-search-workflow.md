# 補助金リサーチ30分を5分にする｜Claude Code × Browser Use CLI で J-Net21 を自動収集する話

## 「今月使える補助金、ざっと出せる？」が地味にしんどい

経営支援や起業サポートをしていると、定期的に聞かれるのが補助金の話です。
「うちで使える補助金、ある？」
「今月締切のやつ一覧でちょうだい」

その都度、J-Net21（中小企業基盤整備機構が運営する支援情報サイト）を開いて、
分野を選んで、地域を選んで、期間を絞って、結果を一件ずつ確認して、Excelに転記して……。
やってみると分かりますが、これが毎回30分くらい吸われます。月に4回やれば2時間。

「この作業、Claude Code に任せられないか」と思って組んでみたら、5分で終わるようになりました。
しかも、検索条件を変えるだけで毎月使い回せる仕組みになっています。

今日はその構築過程と、ハマったところを共有します。

---

## 何を作ったか

ひとことで言うと、

> 「補助金の検索条件をプロンプトで指示したら、Claude Code が J-Net21 を自動でめくって、条件に合う案件だけを Excel にまとめてくれる」

という仕組みです。

例えばこう投げます。

> 「2026年5月に申請可能な、創業支援系の補助金を全国対象で取ってきて」

すると裏側で、

1. Browser Use CLI で J-Net21 の検索ページを開く
2. JavaScript を実行して全ページの一覧を取得（5ページ・131件）
3. 募集期間を正規表現で抽出して、5月にかかるものだけに絞り込む（56件）
4. 色分け付き Excel に出力

ここまでが約5分で完了します。

---

## 使った道具は3つだけ

- **J-Net21 snavi2**：補助金・助成金・融資・セミナーを横断検索できる中小機構のDB。認証不要・公開情報
- **Browser Use CLI**：ブラウザを CLI で操作する OSS。`open` でページを開いて `eval` で JavaScript を流す
- **Claude Code**：全体のオーケストレーション。プロンプトを受けて Python スクリプトを書き、結果を整形する

特別なものは何もありません。J-Net21 は無料、Browser Use はオープンソース、Claude Code の利用料だけ。
1回の実行コストは数十円といったところです。

---

## 一番効いたのは「URLパラメータを完全解読したこと」

実は最初、Claude Code に「ブラウザを操作してくれ」と頼んでも、検索フォームのチェックボックスを1つ1つクリックしようとして、毎回違うページに飛ばされていました。

そこで方針転換しました。

「フォーム操作は諦めて、URLパラメータを直接組み立てる」

`browser-use eval` で J-Net21 の検索ページの DOM を解析してもらったところ、
すべての検索条件が URL パラメータで完全に制御できることが分かりました。

```
category=2          # 補助金カテゴリ
type[]=3            # 補助金種別
field[]=24,27,28… # 分野（複数）
prefecture[]=00    # 全国
period=2            # 募集期間基準（1なら掲載日基準）
```

このパラメータ表が手に入った瞬間、毎月の自動化が確定しました。
分野や地域を変えたければ、パラメータを差し替えるだけです。

---

## ハマったところ3つ

### 1. `browser-use extract` は実装されていなかった

ドキュメントに載っていて使う気満々だったのですが、2026年5月時点で未実装。
代わりに `eval` で JavaScript を直接流して DOM から取り出す形にしました。
結果的にこちらの方が柔軟で、Set を使った重複排除や独自の整形まで自由にできます。

### 2. `wait` コマンドの引数形式が違った

`browser-use wait 3`（3秒待つ）が通らず、シェルの `sleep 2` で代用。
こういう細かい仕様差は実際に触らないと分からないものです。

### 3. URLの startDate / endDate パラメータが効かなかった

「2026年5月にかかるものだけ」を URL で指定したかったのですが、無効。
仕方なく全件を取得した後、Python 側で募集期間を正規表現で抜いてフィルタする形にしました。
正規表現はシンプルにこれ。

```python
re.search(r'募集期間[：:]\s*(\S+?～\S+)', text)
```

J-Net21 が西暦表記で統一されていたので、令和換算など面倒な処理は不要でした。

---

## 出力された Excel が地味に実用的

`openpyxl` で出力していますが、ただ並べるだけではつまらないので、

- 募集締切が近いものから順にソート
- 締切1週間以内は橙、1ヶ月以内は黄、それ以外は白で色分け
- ヘッダーは固定
- 「検索条件」シートを別タブで保存して、後から条件を見返せるように

を入れています。これだけでクライアントに渡したときの納品感が全然違います。

---

## 「掲載日」と「募集期間」の使い分けが地味に大事

J-Net21 には `period=1`（掲載日基準）と `period=2`（募集期間基準）の2モードがあります。

同じ条件で投げても、
- 掲載日基準（直近1ヶ月の新着）：67件
- 募集期間基準（5月に申請可能）：56件

と全く違う結果になります。
「最新情報をキャッチアップしたい」のか「今月使えるものを知りたい」のかで、出すべき結果が変わる。
最初にここを確認しておかないと、出した一覧が要望とズレます。

---

## Claude Code の「スキル」として配布できる形にした

最後に、この一連のワークフローを Claude Code のスキル（`jnet21-subsidy-search`）として
パッケージ化しました。スキルにすると、

- 自然言語のトリガーフレーズで即起動
- 検索条件のテンプレートを呼び出して即実行
- 他の人にも `.skill` ファイルを渡して共有可能

になります。
私のチームメンバーは、もう「補助金一覧出して」と一言投げるだけで Excel が手元に届きます。

---

## このやり方は他にも転用できる

今回は J-Net21 でしたが、同じパターンは認証不要の公開DBなら大体そのまま使えます。
頭の中のリストとしては、

- 中小機構の他DB（セミナー・イベント、海外展開支援）
- 都道府県の補助金ポータル
- 公募情報サイト（jGrants など）
- 業界団体の助成金ページ

あたりはすぐにでも横展開できそうだなと思っています。

「定期的に同じサイトを巡回して、条件に合うものだけ拾って整形する」
この型は、補助金以外にも市場リサーチ、競合調査、人材募集ウォッチなど、
ビジネスのあらゆる定型情報収集にハマります。
