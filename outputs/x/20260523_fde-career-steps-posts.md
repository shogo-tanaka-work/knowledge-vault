# X単発投稿案：FDEとは何か・どうすれば名乗れるか

5パターンの単発投稿案。すべてLinkedIn記事へ誘導する。
投稿時にURL（`[LinkedIn URL]`）を差し替える。

---

## 案A：定義一言型

```
FDE（Forward Deployed Engineer）って結局なんなんだろう、と気になって整理した。AIエンジニア・業務コンサル・アーキテクト・PMを横断する、AI時代の業務実装責任者って表現がいちばんしっくりきた。PoCで終わらせず本番化まで持ちきる役割。詳細はLinkedInに。

→ [LinkedIn URL]
```

---

## 案B：5段階提示型

```
FDEに至るキャリアを5段階で整理した。
1. AIで小さく作れる
2. 業務フローに組み込める
3. 本番運用を回せる
4. コストとリスクを語れる
5. 現場で成果まで持ちきる
最初から目指す肩書きじゃなく、積み重ねの先に到達する職種、という整理がしっくりきた。

→ [LinkedIn URL]
```

---

## 案C：日本の現実解釈型

```
日本ではFDEという肩書きはまだ定着していない。とはいえ業務改善×情シス×SI×AI導入コンサルを横断できる人の需要は確実に伸びている。肩書きがFDEじゃなくても、中身はほぼFDE、という形が現実的だと感じます。詳細はLinkedInに整理しました。

→ [LinkedIn URL]
```

---

## 案D：大企業vs小規模事業者型

```
大企業向けFDEと小規模事業者向けFDEで、求められる能力が全然違う。大企業はCRM/ERP統合とCTO・法務説得。小規模は経営者本人と直接やる仕事で、技術の高度さよりNoCode/SaaSの選定眼が核心になる。後者に強い人材、まだ国内では希少です。

→ [LinkedIn URL]
```

## 運用メモ

- すべてLinkedIn記事への誘導が前提
- 投稿時間の目安：平日の朝7〜8時台、または夜21時台
- ハッシュタグは入れないか、入れても1個まで（例：`#FDE` または `#AIキャリア`）
- 「目指す肩書き」「中身」など、断定よりも自分の整理を共有するトーンを保つ
- 煽り表現・誇張は使わない
- LinkedIn の投稿が公開されてからまとめて連投する運用が自然
