# X（旧Twitter）連投スレッド｜JNET21 補助金リサーチ自動化

## 投稿1（フック・単独でも成立する形）

補助金リサーチ、毎月30分かかってたのを5分にした。

Claude Code + Browser Use CLI で J-Net21 を自動で巡回 → 募集期間で絞り込み → 色分けExcel出力。

131件取って、5月に申請できる56件だけ抜き出すまで、ほぼ無人。

仕組みをぶら下げる👇

---

## 投稿2（背景）

J-Net21（中小機構の補助金DB）は良いんだけど、

・分野チェック
・地域チェック
・期間絞り込み
・1件ずつ確認
・Excelに転記

を毎月やってると地味に時間が溶ける。
月4回で2時間。年間で24時間。これはもう自動化案件。

---

## 投稿3（仕組み）

やってることは3層構造：

① Browser Use CLI でページを開く（`open`）
② JavaScript で DOM を直接抽出（`eval`）
③ Python で募集期間を正規表現で抜いてフィルタ → openpyxl で Excel 化

Claude Code がこの3層をまとめて書いてくれる。

---

## 投稿4（一番の発見）

最大の山は「フォーム操作を諦めたこと」。

チェックボックスをクリックする方針だと毎回違うページに飛ぶ。

代わりに `browser-use eval` で URL パラメータを全解析。
`field[]` `type[]` `period=` まで分かれば、あとは URL を組むだけで全件取れる。

---

## 投稿5（ハマりポイント共有）

ハマりどころ3つ：

・`browser-use extract` は未実装（2026/05時点）→ `eval` で代替
・`wait N秒` の引数形式が違う → シェル `sleep` で代替
・URL の startDate/endDate は無効 → 全件取って Python でフィルタ

ドキュメントに載ってても触ってみないと分からないこと、多い。

---

## 投稿6（成果）

出力 Excel は、

・締切順ソート
・1週間以内は橙、1ヶ月以内は黄で色分け
・ヘッダー固定
・「検索条件」シートを別タブで保存

までやってる。納品感が出る。

Claude Code のスキル化してチームに配布済み。
「補助金一覧出して」の一言で Excel が届く。
