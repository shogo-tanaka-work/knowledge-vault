# 02. 使い方手順書 — 5 つのシナリオ

このドキュメントは「コピペで動く」レベルでの手順書です。コマンドはすべて **プロジェクトルート（`consulting-slide-solution/` 直下）** から実行する前提で書いています。

---

## 0. 事前準備（最初に 1 回だけ）

### 0-1. 必須環境

| 用途 | 必要なもの | インストール確認コマンド |
|---|---|---|
| Claude Code でスキル発動 | Claude Code | `claude --version` |
| PPTX 再生成（HTML 編集後のみ必要） | Python 3.10+ | `python3 --version` |
| 同上 | 依存パッケージ | `pip list \| grep -E "pptx\|playwright\|lxml"` |
| 画像埋込 PPTX 生成 | Google Chrome | `open -a "Google Chrome"` |

### 0-2. Python 依存パッケージのインストール（PPTX 再生成する場合のみ）

```bash
pip install python-pptx playwright Pillow lxml
playwright install chromium    # ブラウザ未インストールの場合
```

### 0-3. Claude Code を起動してスキルが認識されているか確認

```bash
cd /path/to/consulting-slide-solution
claude
```

Claude Code 内で `/skills` を実行し、以下 2 つが表示されれば OK。

- `consulting-slide-remake`
- `slide-design-patterns`

> 表示されない場合は `.claude/skills/` 配下にディレクトリがあるか確認してください。

## シナリオ 一覧

| # | やりたいこと | 起点 | 触るフォルダ |
|---|---|---|---|
| 1 | 完成済みスライドを投影 / 微編集 | 既存 PPTX | `outputs/` |
| 2 | **ゼロから新規スライドを作る** | テキスト指示のみ → 画像生成 AI 経由 | `assets/references/` → `tmp-source/` → `outputs/` |
| 3 | 参考画像から自分のテーマで再現 | 既存スライド画像 | `assets/references/` → `tmp-source/` → `outputs/` |
| 4 | 画像 / PDF を編集可能 PPTX に変換 | 画像 / PDF | `assets/references/` → `outputs/` |
| 5 | HTML 編集して PPTX 再ビルド | 既存 HTML | `tmp-source/` → `outputs/` |

> **共通の前提**: 完成したクライアント向け納品物は **必ず `outputs/` 配下に出ます**。`tmp-source/` はその一歩手前の HTML ソースです。

## シナリオ 1: 完成済みスライドをそのまま投影 / 微編集する（最頻出）

### 1-A. 投影だけしたい

```bash
open outputs/consulting-html-slides.pptx
```

PowerPoint で開いたら投影モードで再生するだけ。

### 1-B. 文言だけ直したい（見た目は維持）

```bash
open outputs/12-ai-workflow-asis-editable.pptx
```

開いた PPTX 上で対象テキストボックスをクリックして直接編集。すべてネイティブテキストボックスなので画像化はされていない。

### 1-C. 見た目ごと変えたい

`tmp-source/*.html` を編集 → **シナリオ 5** に進む。

## シナリオ 2: ゼロから新規スライドを作る（参考画像なし）

### このシナリオの流れ

```
あなたのテーマ文章
    ↓ (画像生成 AI: Nano Banana 等)
構図ラフ画像
    ↓ (assets/references/ に保存)
Claude Code に依頼
    ↓ (consulting-slide-remake Mode A が発動)
構造化プロンプト
    ↓ (Claude Code に「HTML 化して」と追加依頼)
tmp-source/NN-xxx.html
    ↓ (scripts/01_build_pptx_image_embed.py)
outputs/consulting-html-slides.pptx ← 完成
```

### 手順

#### Step 1. テーマと構成をテキストで書き出す

メモ帳でもよい。最低限の項目:

```
- 章番号: 3.2
- メッセージ: AI 浸透における 4 つの障壁
- レイアウト: 4 象限マトリクス（組織/データ/人材/運用）
- 配色: 政府系シンクタンク風（ネイビー＋ブロンズ）
- KEY TAKEAWAY: 左下に1行で結論
```

#### Step 2. 画像生成 AI に投げて構図ラフを 1 枚作る

| AI | 強み |
|---|---|
| Nano Banana | 構図再現性が高い、図解寄り |
| GPT-Image (ChatGPT) | 構図インスピは強いがタイポは弱い |
| Claude Design tool | ベクター傾向、整列が綺麗 |

→ タイポや細部が崩れても **構図さえ取れていれば OK**。

#### Step 3. 生成画像を `assets/references/` に保存

```bash
# 例: ダウンロードした画像を移動
mv ~/Downloads/generated-rough.png assets/references/ai-barriers-rough.png
```

#### Step 4. Claude Code に Mode A を発動させる

Claude Code のプロンプト欄に貼る（**画像はパス参照で渡せます**）:

```
@assets/references/ai-barriers-rough.png をベースに、政府系シンクタンク風で
「AI 浸透における 4 つの障壁」のスライドをリメイクしたい。
テキストは構図ラフのタイポを無視し、Step 1 で書き出した内容に置き換えてください。
```

→ `consulting-slide-remake` Mode A が自動発動し、戦略コンサル風スライド生成プロンプトが出力される。

#### Step 5. そのまま HTML 化を依頼

Mode A の出力を受け取ったら、続けて以下を依頼:

```
このプロンプトの内容を tmp-source/13-ai-barriers.html として作成してください。
assets/style.css を読み込み、既存の tmp-source/*.html と同じ枠（16:9 / 1280×720）
を踏襲してください。
```

→ `tmp-source/13-ai-barriers.html` が新規追加される。

#### Step 6. ブラウザで確認

```bash
open tmp-source/13-ai-barriers.html
```

レイアウト崩れがあれば Claude Code に「ここを直して」と追加依頼。

#### Step 7. PPTX 化（**シナリオ 5** に進む）

新しい HTML が `tmp-source/` に入っているので、シナリオ 5-2 のコマンドを実行すれば `outputs/consulting-html-slides.pptx` に組み込まれる。

## シナリオ 3: 参考スライド画像から、自分のテーマで似たスライドを作る

> 「このマッキンゼー風スライドのレイアウトを真似て、当社の AI 施策を表現したい」というケース。**本ソリューションの中核ユースケース**。

### 手順

#### Step 1. 参考スライドを `assets/references/` に保存

```bash
# 画像化済みのファイルを移動
mv ~/Downloads/mckinsey-sample.png assets/references/mckinsey-sample.png
```

PDF の場合は対象ページを画像（PNG / JPG）に書き出してから保存してください。

#### Step 2. Claude Code に Mode A を発動させる

```
@assets/references/mckinsey-sample.png のスライドをアビーム風で再現したい。
テーマは「AI 提案書作成ワークフローの現状」で、私の組織の営業プロセスに
置き換えてください。章番号は 2.1、KEY TAKEAWAY は「提案書作成の 60% を AI に置換」。
```

#### Step 3. 出力されたプロンプトを HTML 化

```
このプロンプトの内容を tmp-source/14-ai-proposal-workflow.html として作成して。
既存スライドと同じスタイル（assets/style.css、16:9 / 1280×720）を踏襲。
```

#### Step 4. ブラウザで確認 → PPTX 化（シナリオ 5 へ）

```bash
open tmp-source/14-ai-proposal-workflow.html
# 問題なければシナリオ 5-2 のコマンドで outputs/ に焼く
```

### 発動キーワード（Mode A）

「コンサル風スライドを再現」「アビーム風で作って」「マッキンゼー風スライドのリメイク」「画像から再現プロンプト作って」

## シナリオ 4: 既存の画像スライド / PDF を、編集可能 PPTX に変換する

> 「PDF でもらった綺麗なスライドを、PowerPoint で文言だけ直したい」というケース。HTML 中間表現を経由せず、画像 → 直接編集可能 PPTX に変換する。

### 手順

#### Step 1. 変換対象を `assets/references/` に置く

```bash
mv ~/Downloads/client-deck-p3.png assets/references/client-deck-p3.png
```

#### Step 2. Claude Code に Mode B を発動させる

```
@assets/references/client-deck-p3.png をズレなく編集可能な PPTX に変換するための
再構築指示書を作って。文字はすべてネイティブテキストボックス、装飾はシェイプ、
画像埋め込みは禁止でお願いします。
```

→ `consulting-slide-remake` Mode B が自動発動し、座標・色・フォント・テキスト指定一式の **PPTX 再構築指示書** が出力される。

#### Step 3. その指示書を Claude Code に渡して実装させる

```
上記の再構築指示書をもとに、scripts/03_build_client_deck.py として
python-pptx で実装してください。出力先は outputs/client-deck-p3-editable.pptx。
既存の scripts/02_build_pptx_editable.py のコード規約（NAVY / BRONZE 定数の使い方、
add_textbox / add_shape ヘルパーの形）に揃えてください。
```

#### Step 4. 実行して確認

```bash
python3 scripts/03_build_client_deck.py
# → outputs/client-deck-p3-editable.pptx が生成される

# 編集可能性の検証（ppt/media が空なら画像ゼロ＝完全編集可能）
unzip -l outputs/client-deck-p3-editable.pptx | grep "ppt/media"
# → 何も出力されなければ OK
```

```bash
open outputs/client-deck-p3-editable.pptx
```

### 発動キーワード（Mode B）

「これをパワポにして」「画像を編集可能な pptx に」「PDF をズレなく PPTX 化」

## シナリオ 5: HTML スライドを編集して PPTX 再生成する

> シナリオ 2/3 で HTML を追加した直後、または既存 HTML の文言を手で直した直後に必ず通る経路。

### 手順

#### 5-1. HTML を編集（必要な場合のみ）

```bash
# プレビューしながら編集
open tmp-source/12-ai-workflow-asis.html       # ブラウザでプレビュー
code tmp-source/12-ai-workflow-asis.html       # VS Code 等で編集
```

文言を直したい場合は HTML 内の該当テキスト or SVG 内の `<text>` を書き換えるだけ。

#### 5-2. 画像埋込 PPTX を再生成（投影用・最終納品）

```bash
python3 scripts/01_build_pptx_image_embed.py
```

- 入力: `tmp-source/*.html` 全て
- 出力: **`outputs/consulting-html-slides.pptx`**（上書き）
- 中間 PNG: `scripts/.png-cache/`（再生成のたびに上書き）
- 所要時間: 12 スライドで約 30 秒

#### 5-3. 編集可能 PPTX を再生成（クライアントに渡す場合）

```bash
python3 scripts/02_build_pptx_editable.py
```

- 入力: `tmp-source/12-ai-workflow-asis.html` の設計値（**現状 slide 12 専用**）
- 出力: **`outputs/12-ai-workflow-asis-editable.pptx`**（上書き）

> ⚠️ 他のスライドを編集可能 PPTX 化したい場合は、シナリオ 4 と同じ流れで Claude Code に「`scripts/02_build_pptx_editable.py` を参考に slide N 用のスクリプトを作って」と依頼してください。

#### 5-4. 完成確認

```bash
open outputs/consulting-html-slides.pptx                # 画像埋込版
open outputs/12-ai-workflow-asis-editable.pptx          # 編集可能版
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| `claude` 実行で `/skills` にスキルが表示されない | `.claude/skills/` が認識されていない | プロジェクトルートで `claude` を起動しているか確認、`ls .claude/skills/` で実体確認 |
| `ModuleNotFoundError: No module named 'pptx'` | Python パッケージ未インストール | `pip install python-pptx playwright Pillow lxml` |
| `playwright._impl._errors.Error: Executable doesn't exist` | Chrome / Chromium が見えない | `playwright install chromium` |
| `01_build_pptx_image_embed.py` の Chrome パス不一致 | スクリプトが `/Applications/Google Chrome.app/...` を直指定 | macOS 以外の場合は該当行を削除して `playwright install chromium` 経由に変更 |
| 矢印が左上に飛ぶ（編集可能 PPTX） | `add_connector` を delta 座標で呼んでいる | 絶対座標 `(begin_x, begin_y, end_x, end_y)` で渡す（修正済） |
| HTML を編集したのに PPTX が変わらない | スクリプト未実行 | `python3 scripts/01_build_pptx_image_embed.py` を再実行 |
| `outputs/` の中身を確認したい | — | `unzip -l outputs/xxx.pptx \| head` で内部構造を確認できる |

## 参考: ファイル直接編集ツリー

最頻出の編集対象だけ抜粋:

```
tmp-source/12-ai-workflow-asis.html   ← 文言・配色を直接編集
   └─ 中の SVG 内の <text> を書き換えるだけで反映

assets/style.css                          ← 配色テーマを横断的に変える時
   └─ :root 直下のカスタムプロパティ

scripts/02_build_pptx_editable.py         ← 編集可能 PPTX のレイアウト基準
   └─ NAVY / BRONZE 等の定数を変えれば配色変更
```

## まとめ: フォルダの動き方早見

```
[ assets/references/ ]   ← 参考画像・構図ラフを置く（シナリオ 2/3/4 の入口）
         ↓
[ Claude Code + .claude/skills/ ]   ← Mode A / Mode B が発動
         ↓
[ tmp-source/*.html ]   ← Claude or 手で編集する中間ソース
         ↓
[ scripts/*.py を実行 ]
         ↓
[ outputs/*.pptx ]   ← 完成。クライアントに渡すのはここ
```
