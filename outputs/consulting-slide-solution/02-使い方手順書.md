# 02. 使い方手順書 — 4 つのシナリオ

## 共通の前提

- **Claude Code がインストール済み**（`https://claude.com/claude-code`）
- 当フォルダを Claude Code で開く（または `cd` してから `claude` コマンド）
- スキルが認識されているかは Claude Code 内で `/skills` で確認

PPTX の再生成だけしたい場合は Claude Code 不要、Python だけでよい。

---

## シナリオ 一覧

| # | シナリオ | 起点 |
|---|---|---|
| 1 | 完成済みスライドを投影 / 微編集 | 既存 PPTX |
| 2 | **ゼロから作る（参考画像なし）** | テキスト指示のみ → 画像生成 AI 経由 |
| 3 | 参考画像から自分のテーマで再現 | 既存スライド画像 |
| 4 | 画像 / PDF を編集可能 PPTX に変換 | 画像 / PDF |
| 5 | HTML 編集して PPTX 再ビルド | 既存 HTML |

---

## シナリオ 1: 既に作ってあるスライドを投影 / 微編集する（最頻出）

クライアント説明に **そのまま使う** ケース。

### 手順
1. `output/consulting-html-slides.pptx` をダブルクリック
2. PowerPoint で開く（Keynote でも可）
3. 投影モードで再生

### 微編集したい場合
- **見た目維持で文言だけ直したい** → `output/12-ai-workflow-asis-editable.pptx` を開いて該当テキストボックスをクリック・編集
- **見た目ごと変えたい** → `src/*.html` をエディタで開いて編集 → 後述シナリオ 5 で再ビルド

## シナリオ 2 (ゼロから): 参考画像が手元にない場合 — 画像生成 AI を経由する

> 「真似たいスライドの参考が手元にない／全く新しい構図を作りたい」というケース。
> 直接 PowerPoint を AI に作らせると安っぽくなるため、**画像生成 AI でいったん 1 枚作り、それをシナリオ 3 の入力に流す** のが現状のベストプラクティス。

### なぜこの 2 段構えなのか
画像生成 AI（Nano Banana / GPT-Image-2 / Claude Design / Imagen 等）はコンサル風の **構図** を高速に作れるが、出力された PNG は文字・配色・数値の微調整ができない。そこで生成画像を **「自前のたたき台」として消費** し、Mode A で構造抽出 → HTML → PPTX に流すことで、最終的に編集可能な状態まで持ち込む。

### 手順
1. **テーマと構成を文章で書き出す**
   ```
   - 章番号: 3.2
   - メッセージ: AI 浸透における 4 つの障壁を、組織・データ・人材・運用 で図解
   - レイアウト指定: 4 象限マトリクス、各象限に 3-5 のキーワード、左下にKEY TAKEAWAY
   - 配色: 政府系シンクタンク風（ネイビー＋ブロンズ）
   ```

2. **画像生成 AI に投げる**
   - Nano Banana: 構図再現性が高い、図解寄り
   - GPT-Image (ChatGPT): タイポは弱いが構図インスピは強い
   - Claude Design tool: ベクター傾向、整列が綺麗
   - 1 枚出ればよい。タイポや細部が崩れていても **構図さえ取れていれば OK**

3. **生成された画像を保存** し、シナリオ 3 の入力として使う
   ```
   この生成画像をベースに、政府系シンクタンク風で
   「AI 浸透 4 つの障壁」のスライドをリメイクしたい。
   テキストは自分のテーマに置き換えてください。
   
   [生成画像をペースト]
   ```

4. `consulting-slide-remake` Mode A が発動し、構造化されたプロンプトを返す
5. そのまま「これで HTML スライドにして」と Claude Code に追加依頼
6. 完成した HTML をシナリオ 5 の手順で PPTX 化

### 注意
- 画像生成 AI の出力にあるタイポ・無意味な日本語は **Mode A の構造解析時にすべて自分の文言で上書き** されるので無視してよい
- 1 発で構図が決まらない場合、画像生成側のプロンプトを 2-3 回練り直してから入口に流すと品質が安定する

## シナリオ 3: 参考スライド画像から、自分のテーマで似たスライドを作る

> 「このマッキンゼー風スライドのレイアウトを真似て、当社の AI 浸透施策を表現したい」というケース。
> **本ソリューションの中核ユースケース**。

### 手順
1. 参考にしたいスライドを画像（PNG / JPG）で保存
2. Claude Code を起動し、以下のように依頼:

   ```
   この参考スライドをアビーム風で再現したい。
   テーマは「AI 提案書作成ワークフローの現状」で、私の組織の
   営業プロセスに置き換えてください。
   
   [参考スライド画像をペースト]
   ```

3. `consulting-slide-remake` スキル（Mode A）が自動発動
4. Claude が画像を構造解析し、**戦略コンサル風スライド生成プロンプト** を出力
5. 出力されたプロンプトを、好みの画像生成 AI（GPT-Image / Claude Design / nano banana 等）に投げる
6. または、そのまま「この内容で HTML スライドを作って」と Claude Code に依頼

### 発動キーワード（参考）
- 「コンサル風スライドを再現」
- 「アビーム風で作って」
- 「マッキンゼー風スライドのリメイク」
- 「画像から再現プロンプト作って」

## シナリオ 4: 既存の画像スライド / PDF を、編集可能 PPTX に変換する

> 「PDF でもらった綺麗なスライドを、PowerPoint で編集したい」というケース。

### 手順
1. 変換対象の画像 / PDF を用意
2. Claude Code に依頼:

   ```
   この画像をズレなく編集可能な PPTX に変換するための
   再構築指示書を作って。
   
   [画像をペースト]
   ```

3. `consulting-slide-remake` スキル（Mode B）が自動発動
4. Claude が **PPTX 再構築プロンプト** を出力（座標・色・フォント・テキストの指定一式）
5. そのプロンプトを Claude Code 自身に渡し、`python-pptx` で実装させる
6. 生成された .pptx を `unzip -l` で `ppt/media/` が空であることを確認 → 編集可能であることの証

### 発動キーワード
- 「これをパワポにして」
- 「画像を編集可能な pptx に」
- 「PDF をズレなく PPTX 化」

## シナリオ 5: HTML スライドを自分で編集して PPTX 再生成する

> 「文言を直したいので、HTML を編集してから両方の PPTX を作り直したい」というケース。

### 手順

#### 5-1. HTML を編集
```bash
# 例: 12 番のスライドの文言を直す
open src/12-ai-workflow-asis.html  # ブラウザでプレビューしながら
code src/12-ai-workflow-asis.html  # VS Code 等で編集
```

#### 5-2. 画像埋込 PPTX を再生成（投影用）
```bash
cd scripts
/usr/local/bin/python3 01_build_pptx_image_embed.py
# → ../output/consulting-html-slides.pptx が更新される
```

所要時間: 12 スライドで約 30 秒。

#### 5-3. 編集可能 PPTX を再生成（必要なら）
```bash
/usr/local/bin/python3 02_build_pptx_editable.py
# → ../output/12-ai-workflow-asis-editable.pptx が更新される
```

> ⚠️ 注意: `02_build_pptx_editable.py` は **slide 12 専用** にハードコードされている。他のスライドを編集可能 PPTX 化したい場合は、Claude Code に依頼してスクリプト追加が必要（シナリオ 3 と同じ流れ）。

## トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| `playwright._impl._errors.Error: Executable doesn't exist` | Playwright がブラウザを持っていない | スクリプト内の `executable_path="/Applications/Google Chrome.app/..."` を参照、または `playwright install chromium` |
| 矢印が左上に飛んでいる | `add_connector` の引数を delta で渡している | `(begin_x, begin_y, end_x, end_y)` の **絶対座標** で渡す（修正済） |
| HTML スライドの iframe ギャラリーで点線位置がズレる | iframe スケーリングと CSS 計算式の相性 | フルスクリーン (`src/*.html` を直接開く) で確認すれば正常 |
| 編集可能 PPTX で書類アイコンの形が違う | `MSO_SHAPE.FOLDED_CORNER` で近似している | `MSO_SHAPE.FREEFORM` で自前パス描画に差し替え可能（要追加実装） |

