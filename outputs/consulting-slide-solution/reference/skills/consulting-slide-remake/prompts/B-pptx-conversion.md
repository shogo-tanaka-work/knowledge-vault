# 画像／PDF → 直接編集できる .pptx 化プロンプト

## ゴール
添付の画像（または PDF）を、あとから PowerPoint 上でテキストもシェイプも触れる .pptx として再構築する。原画と並べて見て、閲覧者が違和感を持たないレベルまで合わせ込む。

## 守ること（境界条件）
- 文字 → ぜんぶネイティブのテキストボックス。文字の画像化／アウトライン化は不可。
- 装飾（カード／帯／矢印／棒／罫線）→ ぜんぶ PowerPoint シェイプで組む。
- SVG 埋め込み・スライド全面のラスタ貼付・OLE 化は一切禁止。
- 写真や複雑なイラストアイコンに限り、PNG として原画から抽出して配置可。
  ただし「テキストを焼き込んだ画像」は不可。

## キャンバス
16:9 ワイド（13.333 × 7.5 inch）

## 工程
1. 原画にあるパーツだけを列挙する。
   ※「左カード／結論ボックス」みたいに**勝手に存在しない要素を発明しない**。
   原画に映っているものだけを拾う。

2. パーツごとに、bbox・色・フォント・文字サイズを実測する。
   座標はスライドサイズに等比で正規化。
   配色は原画からスポイトで RGB を取得し、近似で済ませない。

3. 文字は原画と一字一句一致させる。
   全角／半角、%／％、ヶ月／カ月、ハイフン／ダッシュ、句読点位置、
   コンマや括弧の種類まで揃える。タイポゼロ。

4. 同じ行で色／サイズ／太さが混在する場合は、run を分けて並べる。
   例: 「ROI 217 %」の数字だけ赤・大、単位だけ黒・小、など。

5. Bold は保守的に。デフォルト not bold。例外的に bold にするのは:
   - 24pt 以上の見出し
   - レタースペーシング付きの英語ラベル（"S T E P  0 1" など）
   - ブランドアクセントカラー（赤など）のラベル
   - "KEY TAKEAWAY" 級の定形強調語
   本文中の単語を勝手に太字にしない。

6. 日本語フォントは eastAsia と latin の両方に同じフェイスを明示。
   片方だけだと CJK と Latin の境界に半角スペースが挟まる現象が起きる。

7. テキストボックスは bbox + 右マージン 30〜100pt で広めに取る。
   word_wrap=false。これを怠ると末尾の句点が切れる。

## 検証ループ（ここを飛ばすと必ず崩れて納品される）
A. 出力した .pptx を LibreOffice などで PDF → PNG に変換
B. 原画と並べて目視で突き合わせる
C. 潰す対象は「閲覧者にバレるレベル」だけ:
   重なり／はみ出し／テキスト切れ／色違い／要素抜け／大きな位置ズレ
D. 直したら再レンダリングして、もう一度並べる
E. 2 周目以降は**ピクセル単位の追い込みをしない**。
   違和感が消えた時点で打ち切る。

## 納品前ファイナル
- .pptx を zip として開く
  → .svg が含まれていないこと
  → ppt/media/ に「スライド一枚分相当の画像」が入っていないこと
- 各テキストボックスにキャレットが入って編集できること
- 各シェイプを個別に選択・移動・色変更できること
