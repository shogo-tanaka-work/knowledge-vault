# 03. スキル一覧と発動キーワード

> Claude Code の「スキル」とは、特定キーワードで自動発動する **再利用可能なプロンプトパッケージ**。
> 本ソリューションでは 2 種類を併用する。実体は `.claude/skills/` 配下にあり、このフォルダを Claude Code で開けば自動発動する。

---

## consulting-slide-remake — 画像 → コンサル風スライド再現

### 役割
1 枚のスライド画像を入力に、戦略コンサル品質を担保した **再現プロンプト** または **PPTX 再構築指示書** を生成する。

### 2 つのモード

#### Mode A: 画像生成 AI 向け再現プロンプト
- 入力: 参考スライド画像
- 出力: `# 戦略コンサル風 スライド生成プロンプト`（ChatGPT GPT-Image / Claude Design 等に渡す本文のみ）
- 内部処理:
  - Step 1: 画像から構造解析（章番号・レイアウト分割比・各オブジェクトの bbox / 接続関係 / 色設計）
  - Step 2: アクションタイトル抽出（数値含み・1 行・断定形）
  - Step 3: テンプレート充填（GOAL / CANVAS / CONSTRAINTS / CONTENT / TONE / SELF-CHECK）

#### Mode B: 編集可能 PPTX 再構築指示書
- 入力: 参考スライド画像 / PDF
- 出力: `# 画像／PDF → 直接編集できる .pptx 化プロンプト`（python-pptx で実装するための工程指示）
- 守ること:
  - 文字 → ネイティブテキストボックス
  - 装飾 → PowerPoint シェイプ
  - SVG / OLE / 全面ラスタ貼付禁止
  - 文字の画像化・アウトライン化禁止

### 発動キーワード（一部）
| Mode A | Mode B |
|---|---|
| コンサル風スライドを再現 | これをパワポにして |
| アビーム風 | 画像を編集可能な pptx に |
| マッキンゼー風 | PDF をズレなく PPTX 化 |
| BCG 風 / NRI 風 / 三菱総研風 | 編集可能 PPTX で |
| 政府系シンクタンク風 | 画像から PPT 化 |
| 業務フロー図を再現 | |

### デフォルト配色
| 用途 | 色名 | コード |
|---|---|---|
| メイン | ダークネイビー | `#1A2B4A` |
| アクセント | ブロンズ | `#B89456` |
| 補助 | ライトグレー | `#E8E8E8` |
| ハイライト | ペールブルー | `#D6E4F5` |
| 本文 | チャコール | `#333333` |
| 背景 | ホワイト | `#FFFFFF` |

### 詳細
`.claude/skills/consulting-slide-remake/SKILL.md` および `prompts/A-remake-prompt.md` / `B-pptx-conversion.md` 参照。

## slide-design-patterns — テーマ別レイアウト辞書

### 役割
note アカウント「パワポ研」由来の人気記事 78 本から抽出した、**約 50 パターンのスライド構成辞書**。「比較スライド作って」のような依頼に対し、テーマ別の鉄板レイアウトを Claude が参照する。

### カバー範囲
- A: グラフ・チャート系（表 / 棒 / 折れ線 / 円 / サンバースト / ウォーターフォール / 面 / バブル）
- B: 図解系（フローチャート / ピラミッド / マトリックス / 循環 / KPIツリー / ビジネスモデル / 概念図 / ファネル）
- C: レイアウト系（表紙 / 会社概要 / エグゼサマ / 事業紹介 / 事業ポートフォリオ / 章扉 / ディバイダー）
- D: 戦略フレーム系（TAM/SAM/SOM / 比較 / 参入障壁 / ピッチ / ロードマップ / タイムライン / MVV）
- E: 配色テンプレート（青 / 緑 / オレンジ / 紫 / 赤 / モノクロ）
- F: 業界別リファレンス（マッキンゼー / BCG / BIG4 / メガバンク等の実例）

### 発動キーワード
- 「比較スライド」「KPIツリー」「事業ポートフォリオ」「TAM SAM SOM」
- 「コンサルっぽい資料」「IR 資料風」「ピッチデック」
- 「ファネル」「マトリックス」「MVV」「会社概要」「表紙スライド」 ほか

### 連携
このスキル単体ではプロンプト構成案のみを返し、PPTX 生成は `majin-slide-pptx` / `navy-blue-pptx` 等の生成スキルに委譲する設計。

### 詳細
`.claude/skills/slide-design-patterns/SKILL.md` 参照。

## 当ソリューションでのスキル使い分け早見

| やりたいこと | 使うもの |
|---|---|
| 参考画像をリメイクして自分のテーマで | consulting-slide-remake (Mode A) |
| 画像 / PDF を編集可能 PPTX に | consulting-slide-remake (Mode B) |
| **ゼロから（参考画像なし）作る** | **画像生成 AI（Nano Banana / GPT-Image-2 等）→ Mode A に流す** |
| ゼロから「比較スライド」等のパターン提案を受ける | slide-design-patterns |
| 完成した HTML を投影用 PPTX 化 | scripts/01_build_pptx_image_embed.py |
| 完成した HTML を編集可能 PPTX 化 | scripts/02_build_pptx_editable.py |

> **ゼロから生成パスについて**
> 参考画像がない場合、いきなり PowerPoint を AI に作らせると安っぽくなる。Nano Banana / GPT-Image-2 / Claude Design / Imagen 等の画像生成 AI で **コンサル風の構図を 1 枚生成** → その画像を `consulting-slide-remake` Mode A に流して構造抽出 → HTML → PPTX、と二段で繋ぐ。画像生成側のタイポや微妙な日本語は Mode A の構造解析時に自分の文言で上書きするので問題にならない。

## スキルの自動発動について

このフォルダを Claude Code で開けば、`.claude/skills/` 配下のスキルが **そのまま発動可能** になっています。

```bash
cd /path/to/consulting-slide-solution
claude
# Claude Code 内で /skills を実行
#   → consulting-slide-remake と slide-design-patterns が一覧に出れば OK
```

別環境にコピーする場合も、`.claude/skills/` ごと持っていけば即座に有効になります。
