Consulting Slide Solution

02. 使い方手順書

Claude Code とスライド再現スキルを使い、参考画像・HTML・PPTXを行き来しながらコンサル品質の資料を作るための運用手順。

目次

共通の前提 シナリオ一覧 1. 既存スライドを投影 / 微編集 2. ゼロから作る 3. 参考画像から再現 4. 画像 / PDFを編集可能PPTXへ 5. HTML編集後にPPTX再生成 トラブルシューティング 参考: ファイル直接編集ツリー

共通の前提

  • Claude Code がインストール済み（https://claude.com/claude-code）
  • 当フォルダを Claude Code で開く、または cd してから claude コマンドを実行する
  • スキルが認識されているかは Claude Code 内で /skills で確認する

PPTX の再生成だけしたい場合は Claude Code 不要。Python だけで実行できる。

シナリオ 1: 既に作ってあるスライドを投影 / 微編集する

クライアント説明に そのまま使う ケース。

手順

  1. pptx/consulting-html-slides.pptx をダブルクリック
  2. PowerPoint で開く（Keynote でも可）
  3. 投影モードで再生

微編集したい場合

  • 見た目維持で文言だけ直したい → pptx/12-ai-workflow-asis-editable.pptx を開いて該当テキストボックスをクリック・編集
  • 見た目ごと変えたい → slides/*.html をエディタで開いて編集 → シナリオ 5 で再ビルド

シナリオ 2: 参考画像が手元にない場合

真似たいスライドの参考が手元にない、または全く新しい構図を作りたいケース。直接 PowerPoint を AI に作らせると安っぽくなるため、画像生成 AI でいったん 1 枚作り、それをシナリオ 3 の入力に流す。

なぜこの 2 段構えなのか

画像生成 AI（Nano Banana / GPT-Image-2 / Claude Design / Imagen 等）はコンサル風の 構図 を高速に作れるが、出力された PNG は文字・配色・数値の微調整ができない。そこで生成画像を 自前のたたき台 として消費し、Mode A で構造抽出 → HTML → PPTX に流す。

手順

  1. テーマと構成を文章で書き出す
- 章番号: 3.2
- メッセージ: AI 浸透における 4 つの障壁を、組織・データ・人材・運用 で図解
- レイアウト指定: 4 象限マトリクス、各象限に 3-5 のキーワード、左下にKEY TAKEAWAY
- 配色: 政府系シンクタンク風（ネイビー＋ブロンズ）
  1. 画像生成 AI に投げる
  • Nano Banana: 構図再現性が高い、図解寄り
  • GPT-Image (ChatGPT): タイポは弱いが構図インスピは強い
  • Claude Design tool: ベクター傾向、整列が綺麗
  • 1 枚出ればよい。タイポや細部が崩れていても 構図さえ取れていれば OK
  1. 生成された画像を保存 し、シナリオ 3 の入力として使う
この生成画像をベースに、政府系シンクタンク風で
「AI 浸透 4 つの障壁」のスライドをリメイクしたい。
テキストは自分のテーマに置き換えてください。

[生成画像をペースト]
  1. consulting-slide-remake Mode A が発動し、構造化されたプロンプトを返す
  2. そのまま「これで HTML スライドにして」と Claude Code に追加依頼
  3. 完成した HTML をシナリオ 5 の手順で PPTX 化

注意

  • 画像生成 AI の出力にあるタイポ・無意味な日本語は Mode A の構造解析時に自分の文言で上書きする
  • 1 発で構図が決まらない場合、画像生成側のプロンプトを 2-3 回練り直してから入口に流すと品質が安定する

シナリオ 3: 参考スライド画像から、自分のテーマで似たスライドを作る

このマッキンゼー風スライドのレイアウトを真似て、当社の AI 浸透施策を表現したい、というケース。本ソリューションの中核ユースケース。

手順

  1. 参考にしたいスライドを画像（PNG / JPG）で保存
  2. Claude Code を起動し、以下のように依頼する
この参考スライドをアビーム風で再現したい。
テーマは「AI 提案書作成ワークフローの現状」で、私の組織の
営業プロセスに置き換えてください。

[参考スライド画像をペースト]
  1. consulting-slide-remake スキル（Mode A）が自動発動
  2. Claude が画像を構造解析し、戦略コンサル風スライド生成プロンプト を出力
  3. 出力されたプロンプトを、好みの画像生成 AI（GPT-Image / Claude Design / nano banana 等）に投げる
  4. または、そのまま「この内容で HTML スライドを作って」と Claude Code に依頼

発動キーワード

  • コンサル風スライドを再現
  • アビーム風で作って
  • マッキンゼー風スライドのリメイク
  • 画像から再現プロンプト作って

シナリオ 4: 既存の画像スライド / PDF を、編集可能 PPTX に変換する

PDF でもらった綺麗なスライドを、PowerPoint で編集したいケース。

手順

  1. 変換対象の画像 / PDF を用意
  2. Claude Code に依頼する
この画像をズレなく編集可能な PPTX に変換するための
再構築指示書を作って。

[画像をペースト]
  1. consulting-slide-remake スキル（Mode B）が自動発動
  2. Claude が PPTX 再構築プロンプト を出力（座標・色・フォント・テキストの指定一式）
  3. そのプロンプトを Claude Code 自身に渡し、python-pptx で実装させる
  4. 生成された .pptx を unzip -l で ppt/media/ が空であることを確認する

発動キーワード

  • これをパワポにして
  • 画像を編集可能な pptx に
  • PDF をズレなく PPTX 化

シナリオ 5: HTML スライドを自分で編集して PPTX 再生成する

文言を直したいので、HTML を編集してから両方の PPTX を作り直したいケース。

5-1. HTML を編集

# 例: 12 番のスライドの文言を直す
open slides/12-ai-workflow-asis.html  # ブラウザでプレビューしながら
code slides/12-ai-workflow-asis.html  # VS Code 等で編集

5-2. 画像埋込 PPTX を再生成（投影用）

cd scripts
/usr/local/bin/python3 01_build_pptx_image_embed.py
# → ../pptx/consulting-html-slides.pptx が更新される

所要時間: 12 スライドで約 30 秒。

5-3. 編集可能 PPTX を再生成（必要なら）

/usr/local/bin/python3 02_build_pptx_editable.py
# → ../pptx/12-ai-workflow-asis-editable.pptx が更新される

注意: 02_build_pptx_editable.py は slide 12 専用 にハードコードされている。他のスライドを編集可能 PPTX 化したい場合は、Claude Code に依頼してスクリプト追加が必要。

トラブルシューティング

症状 原因 対処
playwright._impl._errors.Error: Executable doesn't exist Playwright がブラウザを持っていない スクリプト内の executable_path="/Applications/Google Chrome.app/..." を参照、または playwright install chromium
矢印が左上に飛んでいる add_connector の引数を delta で渡している (begin_x, begin_y, end_x, end_y) の 絶対座標 で渡す
HTML スライドの iframe ギャラリーで点線位置がズレる iframe スケーリングと CSS 計算式の相性 フルスクリーン（slides/*.html を直接開く）で確認すれば正常
編集可能 PPTX で書類アイコンの形が違う MSO_SHAPE.FOLDED_CORNER で近似している MSO_SHAPE.FREEFORM で自前パス描画に差し替え可能

参考: ファイル直接編集ツリー

最頻出の編集対象だけ抜粋。

slides/12-ai-workflow-asis.html         ← 文言・配色を直接編集
   └─ 中の SVG 内の <text> を書き換えるだけで反映

assets/style.css                         ← 配色テーマを横断的に変える時
   └─ :root 直下のカスタムプロパティ

scripts/02_build_pptx_editable.py        ← 編集可能 PPTX のレイアウト基準
   └─ NAVY / BRONZE 等の定数を変えれば配色変更
