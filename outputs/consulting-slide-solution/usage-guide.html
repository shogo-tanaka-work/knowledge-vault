Consulting Slide Solution

02. 使い方手順書

Claude Code とスライド再現スキルを使い、テンプレート・参考画像・図解生成を起点に、編集可能なPPTXまでつなげるための運用手順。

目次

共通の前提 判断フロー 1. 参考画像・既存テンプレから再現 2. テキスト指示から構成案を作る 3. 画像生成AIで構図ラフを作る PPTX出力の考え方 トラブルシューティング

共通の前提

  • Claude Code がインストール済み（https://claude.com/claude-code）
  • 当フォルダを Claude Code で開く、または cd してから claude コマンドを実行する
  • スキルが認識されているかは Claude Code 内で /skills で確認する

判断フロー

参考画像・既存テンプレがあるか？

近いPPTX、HTML、スクリーンショットがある場合は、それを構造解析の起点にする。

→

YES: 1. 参考画像・既存テンプレから再現

構図・配色・図解要素を踏襲し、テキストやラベルを差し替える。

NO: テキストだけで構成できるか？

比較表、ロードマップ、2x2、シンプルな業務フローならそのまま構成案から作る。

→

YES: 2. テキスト指示から構成案を作る

パターン辞書を使い、HTML/CSS/SVGでスライド化する。

複雑な図解で、構図の当たりが必要か？

密な業務フロー、概念図、接続関係が多い図は、先に構図ラフを作ると安定する。

→

YES: 3. 画像生成AIで構図ラフを作る

生成画像を「参考画像」として扱い、最終的には1番の再現ルートへ戻す。

1. 参考画像・既存テンプレから再現する

いつ使うか: すでに近い見た目のPPTX、HTMLスライド、参考スライド画像がある場合。構図・余白・配色・図解要素を踏襲し、中身を差し替える。

入力データ

  • 参考画像（PNG / JPG）または既存PPTX
  • 過去に作ったHTMLスライド
  • 差し替えたい本文・章番号・メッセージ・表データ

操作手順

  1. このディレクトリを Claude Code で開く
  2. 参考画像を貼り付ける。PPTXを使う場合は、必要なページを画像化して渡すか、PPTX内の対象ページを指定する
  3. 再現したいテイスト、差し替えるテーマ、変更したいテキストをプロンプトで指示する
  4. consulting-slide-remake Mode A で構造を抽出する
  5. 抽出結果をもとに slides/*.html を作成または更新する
  6. 必要に応じて scripts/01_build_pptx_image_embed.py または scripts/02_build_pptx_editable.py でPPTX化する

アウトプット先

  • HTMLスライド: slides/*.html
  • 画像埋込PPTX: pptx/consulting-html-slides.pptx
  • 編集可能PPTX: pptx/*-editable.pptx または検証用の別名PPTX

プロンプト例（参考画像を使う）

この参考スライドを政府系シンクタンク風で再現したい。
テーマは「AI 提案書作成ワークフローの現状」で、私の組織の
営業プロセスに置き換えてください。

[参考スライド画像をペースト]

プロンプト例（既存PPTXを使う）

このPPTXの12ページ目の構図をベースに、
文言を「AI活用ロードマップ」に差し替えてください。
配色、章番号、KEY TAKEAWAY、右側の説明欄の型は維持し、
HTMLスライドと編集可能PPTXを作成してください。

向いている場面

  • トンマナや余白設計を既存資料に寄せたい
  • 章番号、フッタ、KEY TAKEAWAYなどの型を維持したい
  • 構図はそのまま使い、テキスト・数値・ラベルだけ差し替えたい
  • ボックス数の増減や配置の微調整だけで済む

2. テキスト指示から構成案を作る

いつ使うか: 参考にできる見た目がなく、テーマ・論点・構成メモだけがある場合。まずスライドの骨格を作り、シンプルな図解ならそのままHTML化する。

入力データ

  • 作りたいテーマ
  • 伝えたい結論、章番号、想定読者
  • 使いたいフレーム（比較表、ロードマップ、KPIツリー、業務フローなど）

操作手順

  1. このディレクトリを Claude Code で開く
  2. テーマ、結論、使いたいフレームをプロンプトで渡す
  3. slide-design-patterns でレイアウト候補を出す
  4. アクションタイトル、主要メッセージ、図解パターンを決める
  5. slides/*.html としてHTML/CSS/SVGでスライドを作る
  6. ブラウザまたはスクリーンショットで密度・整列・読みやすさを確認する
  7. 複雑な図解になりそうなら、3番で構図ラフを作ってから1番へ戻す

アウトプット先

  • 構成案: Claude Codeの応答本文
  • HTMLスライド: slides/*.html
  • PPTX化する場合: pptx/*.pptx

プロンプト例

「AI活用ロードマップ」を1枚で説明したい。
構成は、現状課題 → 重点施策 → 導入ステップ → 効果指標。
政府系シンクタンク風のネイビー＋ブロンズで、
HTMLスライドとして作成してください。

向いている場面

  • まだ見た目の正解が決まっていない
  • まず論理構成や見せ方を探索したい
  • 比較表、ロードマップ、2x2、単純なプロセス図などで足りる
  • 複数案を出してから1案に絞りたい

3. 画像生成AIで構図ラフを作る

いつ使うか: 参考画像がないが、複雑な業務フロー・概念図・構造図が必要な場合。画像生成AIで作るのは完成物ではなく、1番の再現ルートへ渡すための構図ラフ。

なぜ構図ラフを挟むのか

テキストだけで密な業務フローを一発生成すると、接続関係や情報密度が崩れやすい。画像生成AIで先に構図の当たりを作り、その画像を参考画像として consulting-slide-remake に渡すと、1番の「参考画像から再現」ルートに持ち込める。

入力データ

  • 図解したい業務・概念・システム構成の説明文
  • 登場要素（担当者、DB、帳票、CSV、処理ステップなど）
  • 希望するテイスト（政府系シンクタンク風、アビーム風、ネイビー＋ブロンズなど）

操作手順

  1. 図解したい対象を文章で整理する
  2. 画像生成AIに、文字少なめ・図形中心で構図ラフを作らせる
  3. このディレクトリを Claude Code で開く
  4. 生成画像を参考画像として Claude Code に渡す
  5. 1番と同じく consulting-slide-remake Mode A で構造を抽出する
  6. 本文・ラベル・数値を正しい内容に置き換えて slides/*.html としてHTML化する
  7. 必要に応じて編集可能PPTXへ変換する

アウトプット先

  • 構図ラフ: 画像生成AIから出力されるPNG / JPG
  • HTMLスライド: slides/*.html
  • 編集可能PPTX: pptx/*-editable.pptx

画像生成AIに投げるプロンプト例

AI提案書作成ワークフローを、2レーンの業務フロー図として可視化してください。
左から右に処理が進み、DB、CSV、帳票、営業担当への引き渡しが分かる構図。
文字は最小限で、ネイビー、ブロンズ、ライトグレーを基調にしてください。

注意

  • 画像内の日本語や細かい数字は採用せず、後段で正しい文言に置き換える
  • 生成画像は完成物ではなく、参考画像を人工的に作るためのたたき台として扱う
  • DBアイコン、書類、矢印、番号バッジは最終的にHTML/SVGまたはPowerPoint図形で再構築する

PPTX出力の考え方

出力方式 特徴 必要なもの 使いどころ
HTML → PNG → PPTX HTMLをブラウザ表示どおりPNG化し、PPTXに全面貼付する。PowerPoint上では画像扱い。 Playwright + Chrome。ローカルHTMLを正確にレンダリングするためで、スクレイピング用途ではない。 見た目の再現性を優先したい場合。
HTML設計値 → 編集可能PPTX python-pptxでテキスト・図形をPowerPoint要素として再構築する。細部は近似になる。 python-pptx。この方式ではPlaywrightは使わない。 後から文言・色・位置を直したい場合。

再生成コマンド

cd scripts
/usr/local/bin/python3 01_build_pptx_image_embed.py

/usr/local/bin/python3 02_build_pptx_editable.py

02_build_pptx_editable.py は現状 slide 12 専用。別スライドを編集可能PPTX化する場合は、対象スライドごとに座標・図形・テキストを再構築する。
