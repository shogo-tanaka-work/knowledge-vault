# LinkedIn投稿｜中小企業支援の現場で「補助金リサーチ30分→5分」を実現した話

中小企業支援や起業サポートに携わる方であれば、今月使える補助金を一覧化するという地味で頻度の高い作業に時間を奪われた経験があるのではないでしょうか。

私自身、月に数回発生するこの作業に毎回30分前後を費やしていました。J-Net21を開き、分野・地域・期間を切り替え、一件ずつ確認してExcelに転記する。クライアントごとに条件を変えれば、それだけで半日が消える日もあります。

この作業をAIエージェントのClaude Codeとブラウザ自動化ツールのBrowser Use CLIで再構築したところ、所要時間は5分にまで短縮できました。

**【実現したこと】**

検索条件を自然言語で指示すると、以下が一気通貫で実行されます。

1. J-Net21の検索ページを自動で巡回
2. 全件取得後、募集期間に基づいてフィルタリング
3. 締切が近いものから順にソートし、色分け付きのExcelで出力

実際の検証では、131件の補助金から「2026年5月に申請可能な56件」を抽出し、納品可能な品質のExcelに整形するまで約5分でした。

---

**【ビジネスインパクト】**

定量的な効果は以下のとおりです。

- 工数: 30分/回 → 5分/回（▲25分）
- 月次想定（週1回）: 100分/月の工数削減
- 人件費換算（時給6,000円）: 月10,000円超のコスト削減
- 初期投資: 実質ゼロ。J-Net21は無料、Browser Use CLIはOSS

本当の価値はコスト削減そのものよりも、リサーチ品質の均質化にあると考えています。手作業では条件の解釈ブレが起こりやすく、担当者によって出てくる一覧が微妙に違うことがあります。同じパラメータで自動実行すれば、誰がやっても同じ結果が出る。クライアントへの説明責任という意味でも、これは大きな前進です。

---

**【再現性について】**

この仕組みは認証不要の公開DBであれば応用範囲が広く、以下のような業務全般にそのまま転用できます。

- 都道府県の補助金ポータル
- jGrantsなどの公募情報サイト
- 業界団体の助成金情報
- 市場調査や競合動向のウォッチ

経営支援の現場でAIを活用するというと、つい派手な事例に目が行きがちですが、実務で効くのはこうした地味で頻度の高い作業を確実に潰す使い方だと感じています。
