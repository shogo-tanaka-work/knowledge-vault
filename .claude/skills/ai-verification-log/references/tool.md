# ツール単位 検証ログ — 処理フロー＆テンプレート

特定ツール1つを深掘り評価するための記録。機能・コスト・リスク・比較など多角的な評価を管理する。

---

## コマンド別処理フロー

### 🟢 init（新規作成）

1. ファイルパスを決定する（指定がなければ `structured/tools/YYYYMMDD_[ツール名].md`）
2. 既存ファイルがある場合は上書きせずユーザーに確認する
3. 以下の雛形テンプレートを書き出す
4. 完了メッセージを出す：`✅ 雛形ファイルを作成しました。パス: [パス]`

### 🔄 update（差分追記）

1. 対象ファイルを読み込む（パス未指定なら `structured/tools/` 内の最新ファイルを候補提示）
2. 入力メモを前処理してセクションに振り分ける（下の「振り分け表」参照）
3. 既存内容は削除せず追記する（テーブルは行を追加、箇条書きは末尾に追記）
4. プレースホルダー（`（要記入）` `（随時更新）`）はメモが入ったら置き換える
5. ファイル冒頭の `最終更新:` を今日の日付に書き換える
6. ファイル末尾の更新ログに `YYYY/MM/DD | [更新内容の概要]` を追記する
7. ファイルを上書き保存する
8. 完了メッセージ：`✅ ログを更新しました。パス: [パス] / 更新箇所: [セクション名]`

### 🏁 finalize（最終化）

1. ファイルを読み込んで空欄箇所を把握する
2. 検証ステータスを `検証中` → `完了` に変更する
3. 検証期間の終了日を今日の日付に更新する
4. 10. 判定と今後のアクションの導入判定チェックボックスを確認・整理する
5. 残っている空欄を `（情報なし）` に統一する
6. 更新ログに `YYYY/MM/DD | 最終化（finalize）` を追記する
7. ファイルを上書き保存する
8. 残っている空欄セクションをユーザーに報告し、Note/SNS用サマリー生成の要否を確認する

## publishモード：記事ドラフト自動生成

finalizeモード後、またはユーザーが「記事にして」「Note記事を作って」等で明示的に要求した場合に実行する。
**処理の全体フローは SKILL.md の Step 2-A に従う。** ここではツール単位の記事テンプレートのみ定義する。

published_url: ""
