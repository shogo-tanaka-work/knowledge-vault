# 予約・売上情報の自動スクレイピング検証

ステータス: 保留

**カテゴリ:** ツールPoC（データ収集）

**期間:** 2025/07/26 - 2025/08/01

**プロジェクトステータス:** 中断（法的・倫理的リスク考慮）

### 1. 背景と目的

- **開始理由:** ホットペッパー等の管理画面から情報を自動取得したいという強い現場ニーズがあるため。
- **期待される効果:** 定型タスク（予約確認・売上集計）の自動化による工数削減。

### 2. 取り組み内容

- **使用ツール・技術:** Claude Desktop, Playwright MCP
- **検証内容:** 特定サイトにログインし、情報を取得するためのプロンプトおよびスクリプトの作成。

### 3. 進捗と成果

- **定性的な成果:** 技術的にはPlaywright MCPを用いることで動的サイトの操作が可能であることを確認。
- **判断:** しかし、利用規約および法的措置の事例を調査した結果、積極的な推奨は避けるべきとの結論に至った。

### 4. 学びとナレッジ

- **重要な気づき:** 技術的な「可能」とビジネス上の「許可」は別物。特に大手プラットフォームの規約は厳格であり、抵触した場合の社会的リスクが高い。

### 5. 課題と対応

- **発生した課題:** ホットペッパー約款（第14条10項）におけるスクレイピング禁止条項。
- **対応方法:** 無理な自動化はせず、公式APIの有無の再確認、またはブラウザ操作を支援する程度の「半自動化」に留める。

### 6. 今後の展開

- **長期的な改善案:** 社内事例としては「技術検証におけるリスクマネジメントの好例」として共有する。
