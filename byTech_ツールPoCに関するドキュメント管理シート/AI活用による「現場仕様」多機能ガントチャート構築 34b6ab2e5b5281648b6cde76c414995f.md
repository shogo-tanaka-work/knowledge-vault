# AI活用による「現場仕様」多機能ガントチャート構築

ステータス: 完了

**カテゴリ:** 制作テーマ別（プロジェクト管理・Excel）

**期間:** 2025/08/16 - 2025/08/22

**プロジェクトステータス:** 完了（運用ロードマップ確立）

### 1. 背景と目的

- **開始理由:** 標準的なExcelガントチャートテンプレートは、土日休み固定のものが多く、建設・飲食業界のような不定休や現場独自の稼働形態に対応できないため。
- **解決したい課題:** 「非営業日（不定休）の動的な取り扱い」「親子タスクの連動」「進捗率の視覚化（色の濃淡）」をすべて満たし、かつ社内で広く使えるExcelベースの管理ツールを構築する。

### 2. 取り組み内容

- **実施施策:** ChatGPT（チャッピー）との対話を通じ、複雑なExcel関数のデバッグを繰り返しながらプロトタイプを多段階（v1〜v7）で開発。
- **使用ツール・技術:** Microsoft Excel (INDEX/MATCH, AGGREGATE関数), ChatGPT
- **主要な意思決定:** Excel関数の限界やバージョンの差異（Mac版16.98基準）を考慮し、XLOOKUP等の新関数を使わず、堅牢な旧関数のみでロジックを組む「フェーズ分けロードマップ」を策定した。

### 3. 進捗と成果

- **達成できたこと:** 不定休・複数従業員・親子タスクを1枚で管理でき、進捗率に合わせてセル内の色が変化する「進捗オーバーレイ付きガントチャート」の完成。
- **定性的な成果:** 「一気に作らせるのではなく、前提を固定してから段階的に指示を出す」という、複雑なExcelシートをAIに量産させるための独自の指示テンプレートを確立した。

### 4. 学びとナレッジ

- **うまくいったこと（Good）:** 業界ごとの稼働状況（建設・飲食等）に対応させるため、「稼働日管理シート」を分離し、フラグ管理で柔軟に反映させる設計にできた点。
- **改善ポイント（Improve）:** GPTsでの量産化を試みたが、プルダウンの欠落や関数の不適用が多く、最終的な仕上げには依然として人間のチューニングが必要である。

### 5. 課題と対応

- **発生した課題:** 親タスクのみの管理では不十分であり、現場では「子タスク」の進捗が親に反映されるUIが必須であった。
- **対応方法:** 親子関係を定義する「キー（担当者◆親タスク）」を設定し、AGGREGATE関数で子の開始/終了日の最小・最大値を自動取得するロジックを実装した。

### 6. 今後の展開

- **次のアクション:** 確立した「フェーズ0〜5」のロードマップを標準マニュアル化し、クライアントごとのカスタマイズ依頼に即応できる体制を作る。
- **長期的な改善案:** スマートシート（Smartsheet）のような高度なUIを目指し、Googleスプレッドシートのタイムライン機能を用いた代替案との比較検証を行う。
