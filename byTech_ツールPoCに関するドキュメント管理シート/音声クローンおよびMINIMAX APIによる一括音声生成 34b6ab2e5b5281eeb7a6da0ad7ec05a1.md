# 音声クローンおよびMINIMAX APIによる一括音声生成

カテゴリ: 音声系
ステータス: 進行中

**カテゴリ:** ツールPoC（音声AI・コンテンツ制作）

**期間:** 2026/01/20 - 2026/02/04

**プロジェクトステータス:** プロトタイプ実装完了・量産体制移行中

### 1. 背景と目的

- **開始理由:** 動画制作（Remotion等）やSNS投稿において、特定のキャラクター（テクじい等）の声を一貫して使用したいが、ブラウザ上のGUIツールで1つずつ生成するのは作業工数が膨大になるため。
- **解決したい課題:** 台本の修正に伴う音声の即時・大量生成、およびAPI経由での「声の高さ・抑揚」の自動制御による、人間味のあるナレーションの実現。

### 2. 取り組み内容

- **実施施策:** 1. MINIMAX APIの音声クローン機能（T2A: Text to Audio）の技術検証。 2. 複数のテキスト台本を一括で音声ファイル（MP3/WAV）として出力するPythonスクリプトの開発。 3. 特定の人物（テクじい等）の音声素材を用いたクローニング精度の確認。
- **使用ツール・技術:** MINIMAX API, Python, Claude Code

### 3. 進捗と成果

- **達成できたこと:** テキストファイルを読み込ませ、API経由で一括して音声ファイルを指定ディレクトリへ出力する基盤の構築。
- **定性的な成果:** ブラウザ操作を介さないことで、動画1本分（数十個のカット）の音声を数秒で用意できる環境を確立。クローン音声の「キャラクターの再現性」が高嶋氏のチェックを通過できるレベルであることを確認。

### 4. 学びとナレッジ

- **技術的な発見:** MINIMAX APIは他社の音声合成と比較しても日本語の自然なイントネーションに強く、かつAPIのレスポンスが高速であるため、大量生成に向いている。
- **改善ポイント:** 感情表現やポーズ（間）の入れ方は、APIパラメータ（speed, pitch等）や特定のタグ挿入で制御可能。

### 5. 課題と対応

- **発生した課題:** API利用のための組織用APIキーの発行・管理体制。
- **対応方法:** 現時点では個別の検証用キーで対応しているが、本番量産に向けてCSアカウントでの一元管理および請求管理への統合を申請中。

### 6. 今後の展開

- **次のアクション:** Remotion（動画生成エンジン）とのAPI連携により、「台本入力 → 音声生成 → 動画合成」を全自動で行うパイプラインの構築。
- **長期的な改善案:** ユーザーの反応（聞き取りやすさ等）をフィードバックし、最適な「声のトーン」をAIが自動選択する学習機能の検討。
