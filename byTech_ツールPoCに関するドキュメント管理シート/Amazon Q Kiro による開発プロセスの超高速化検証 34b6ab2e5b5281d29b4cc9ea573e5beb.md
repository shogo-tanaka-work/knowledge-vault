# Amazon Q/Kiro による開発プロセスの超高速化検証

ステータス: 完了

**カテゴリ:** ツールPoC（システム開発・AIエージェント）

**期間:** 2025/08/26 - 2025/08/28

**プロジェクトステータス:** 完了（劇的な工数削減を達成）

### 1. 背景と目的

- **開始理由:** Amazon Q/Kiroのウェイトリスト突破に伴い、要件定義から設計・実装タスク生成までをAIがどの程度一貫して担えるか確認するため。
- **期待される効果:** エンジニアが数日を費やす「設計〜環境構築」のフェーズを数分に短縮し、開発全体のROIを最大化する。

### 2. 取り組み内容

- **実施施策:** 食事管理アプリ（あすけん模倣）をテーマに、Kiroによるスペックファイル生成、およびCursorへの引き継ぎによる「バイブコーディング」を実施。
- **使用ツール・技術:** Amazon Q/Kiro (AWS Builder ID), Cursor, SQLite
- **主要な意思決定:** Kiroに実装まで丸投げせず、「仕様書作成（Kiro）」→「コード実装（Cursor Rules適用）」というハイブリッドな役割分担を定義した。

### 3. 進捗と成果

- **定量的な成果:** 作業時間削減率 90%以上。従来2〜3日を要する工程（要件定義〜初期実装）を1時間強で完遂。
- **定性的な成果:** Kiroが生成する`tasks.md`（実装手順書）が極めて具体的で、実装時の迷いが一切なくなるという驚異的な体験を確認した。

### 4. 学びとナレッジ

- **技術的な発見:** Kiroは個人情報保護法やGDPR対応など、法的なポリシーを考慮した設計を自動で行う点が極めて強力。
- **うまくいかなかったこと（Bad）:** ポン出しのコードではUIのバグやSQLiteの接続エラーが発生。AIの「提案」を人間が即座に修正する「バイブコーディング」のスキルが必須。

### 5. 課題と対応

- **発生した課題:** スキーマやパラメータが英語で出力され、国内向け開発ではその後の修正が煩雑になる。
- **対応方法:** Claudeに設計書を読み込ませ、「日本語変換ロジックを自動実装するプロンプト」を実行することで、技術面（英語）と表示面（日本語）の整合性を瞬時に解決した。

### 6. 今後の展開

- **次のアクション:** DB接続（Supabase/Clerk）を含めた認証周りの自動化検証。
- **ビジネスへの応用:** クライアントとの打ち合わせ中に、Kiroで要件を詰め、その場でプロトタイプを見せながら仕様を固める「即興要件定義」の手法を開発する。
