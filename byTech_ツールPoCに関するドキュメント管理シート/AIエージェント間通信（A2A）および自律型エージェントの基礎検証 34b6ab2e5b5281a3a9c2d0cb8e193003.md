# AIエージェント間通信（A2A）および自律型エージェントの基礎検証

ステータス: 完了

**カテゴリ:** ツールPoC（次世代自動化・AIエージェント）

**期間:** 2025/10/11 - 2025/10/17

**プロジェクトステータス:** 完了（基本動作確認済み）

### 1. 背景と目的

- **開始理由:** 単一のAIではなく、複数の役割を持つAIエージェント同士がHTTPS通信等で対話・連携してタスクを完遂する「A2A（Agent to Agent）」の実現可能性を探るため。
- **解決したい課題:** 人間が介在せずに、エージェントが自ら必要な専門エージェントを呼び出し、タスクを完遂する高度な自動化の仕組み化。

### 2. 取り組み内容

- **実施施策:** GitHub Codespace環境にて「Mastra」等のフレームワークを利用。クライアント側から俳句エージェントを呼び出し、特定のテーマに沿った出力を得る検証。
- **使用ツール・技術:** Mastra, Python, GitHub Codespace, A2A Protocol

### 3. 進捗と成果

- **達成できたこと:** エージェントが利用可能な他エージェントを自動発見（discovered_agents）し、メッセージを送信して成果物（俳句）を受け取る一連のフローを確認。
- **定性的な成果:** Strandsというゲームのテーマに対し、AIが「糸・繋がり」を詩的に表現した高品質な回答を生成。エージェント間の「役割分担」の有効性を確認した。

### 4. 学びとナレッジ

- **技術的な発見:** ローカル環境であっても、HTTPS通信を介したエージェント同士の疎通が確立できれば、将来的に異なるプラットフォームやサーバー間のAI連携が可能になる。

### 5. 課題と対応

- **発生した課題:** エージェント間のコンテキスト共有の深度。
- **対応方法:** 今回は単発の呼び出しであったが、複雑なタスクでは状態（State）を保持するための設計が必要になると想定。

### 6. 今後の展開

- **次のアクション:** DifyやAWS Bedrockで構築したエージェント同士を外部APIとして連携させ、実務タスク（要件定義→設計）を分担させる検証。
