## このレッスンのゴール

  

Mac のターミナルから `claude` コマンドで Claude Code を起動できる状態を作ります。最後に、自分の指示で AI がファイルを作る体験を1回踏みます。

  

---

  

## Step 1. ターミナルを開く

  

`Cmd + Space` で Spotlight（Mac の検索ウィンドウ）を開き、`Terminal` と入力して Enter キー。macOS 標準の Terminal.app が立ち上がります。

  

> 📸 [SCREENSHOT-01] Spotlight に「Terminal」と入力した状態のスクリーンショット

> *キャプション例：Cmd + Space を押すと画面中央に検索ボックスが出る。Terminal と入力すると候補が出るので Enter*

  

iTerm2 など別のターミナルを使っている方はそちらでも構いません。

  

---

  

## Step 2. Claude Code をインストールする

  

ターミナルで次のコマンドをそのままコピペして Enter キー。

  

```bash

curl -fsSL https://claude.ai/install.sh | bash

```

  

数十秒〜1分でインストールが完了します。インストール先は `~/.local/bin/claude` です（`~` は自分のホームディレクトリの省略記号）。

  

> 📸 [SCREENSHOT-02] インストール完了時のターミナル出力例

> *キャプション例：Installation complete のような完了メッセージが表示される*

  

公式 `claude.ai` ドメインから取得しているスクリプトです。`curl | bash` が不安な方は、`curl -fsSL https://claude.ai/install.sh -o install.sh` で先にファイルとして取得し、内容を確認してから `bash install.sh` を実行することもできます。

  

> 💡 **なぜこのインストール方法を使うのか**

> Claude Code には他にも `npm install -g @anthropic-ai/claude-code` などの方法がありますが、本コースではこの Native Installer 方式を主軸にしています。Node.js のバージョン違いで動かない事故を避けやすく、自動更新も走るため、初学者がつまずきにくいからです。すでに npm 版で動かしている方はそのままでも大丈夫です。

  

---

  

## Step 3. 動作確認と PATH を通す

  

新しいターミナルウィンドウを開き直して、次を実行します。

  

```bash

claude --version

```

  

バージョン番号が表示されれば成功。Step 4 に進んでください。

  

### `command not found: claude` と出た場合

  

PATH（コマンドを探しに行く場所のリスト）が通っていません。次のコマンドで通します。

  

```bash

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc

source ~/.zshrc

```

  

> 📸 [SCREENSHOT-03] `command not found: claude` のエラー画面

> *キャプション例：このエラーが出ても落ち着いて。PATH を通せば解決する*

  

`source` は設定ファイルをいま開いているターミナルに反映し直すコマンドです。これで `claude --version` が動くようになります。

  

> bash を使っている方は `~/.zshrc` を `~/.bash_profile` に置き換えてください。

## Step 4. Git を確認する

  

```bash

git --version

```

  

`git version 2.x.x` のように表示されれば OK です。

  

表示されない場合は次のコマンドを実行すると、Xcode Command Line Tools のインストール画面が自動で立ち上がります。

  

```bash

xcode-select --install

```

  

ダイアログが開いたら「インストール」をクリック → 使用許諾に「同意する」 → インストール完了まで数分待ってください。完了後、ターミナルに戻って改めて `git --version` を実行し、バージョンが表示されれば OK です。

  

> 📸 [SCREENSHOT-GIT-MAC] Xcode Command Line Tools のインストール確認ダイアログ

> *キャプション例：「インストール」を選択。「Xcode を入手」ではないので注意*

  

これで Mac での準備は完了です。

## Step 5. ログイン・認証

  

Claude Code を使うには claude.ai の Pro / Max プラン、または Anthropic Console の API クレジットが必要です。**無料プランでは Claude Code は使えません**。

  

このコースでは個人開発を想定して claude.ai Pro プランを推奨します。

  

### ログイン手順

  

ターミナルで `claude` と打って起動するだけです。

  

```bash

claude

```

  

初回起動時に自動でブラウザが開き、claude.ai のログイン画面に遷移します。サインイン後、ターミナルに戻れば認証完了です。

  

> 📸 [SCREENSHOT-06] claude.ai のログイン画面（ブラウザ）

> *キャプション例：すでに claude.ai にログイン済みのブラウザなら、確認画面だけ出ることが多い*

  

ブラウザ側にログインコード（短い文字列）が表示されるパターンもあります。その場合はターミナルに `Paste code here if prompted` のような入力欄が出ているはずなので、そこにコードを貼り付けて Enter で完了します。

  

> 📸 [SCREENSHOT-07] ターミナルにログインコードを貼り付ける画面

> *キャプション例：Paste code here if prompted の行にカーソルがある状態*

  

> 📸 [SCREENSHOT-403] Pro/Max プラン未加入時に表示される 403 エラー画面

> *キャプション例：認証が通っても 403 が返る場合はプラン未加入。最初に詰まりやすい箇所なので、claude.ai 側の加入状況を確認*

  

### 認証後の自己診断

  

ログインできたら、続けて次を実行します。

  

```bash

claude doctor

```

  

PATH・バージョン・認証状態をまとめて点検してくれます。すべて緑のチェックなら次に進んでください。

  

### よくある罠：環境変数 `ANTHROPIC_API_KEY`

  

過去に Anthropic API を使っていた方は、シェルの設定ファイルに `ANTHROPIC_API_KEY` が残っていることがあります。これが設定されていると、Pro/Max プランの認証より **API キー課金が優先されます**。意図しない請求につながるので、必ず確認してください。

  

```bash

# 確認

echo $ANTHROPIC_API_KEY

  

# 何か出た場合、いまのターミナルだけ一時的に解除

unset ANTHROPIC_API_KEY

```

  

恒久的に外したい場合は `~/.zshrc` から該当行を削除します。

## Step 6. 最初の Hello

  

ホームディレクトリに練習用フォルダを作って、そこで Claude Code を起動します。

  

```bash

mkdir ~/cc-practice

cd ~/cc-practice

claude

```

  

起動すると対話モード（REPL：1行ずつ命令して結果が返ってくる対話画面）に入ります。

  

> 📸 [SCREENSHOT-08] Claude Code REPL の起動直後の画面

> *キャプション例：プロンプト記号と一緒に入力待ち状態になっている*

  

プロンプトに対して話しかけてみましょう。英語で `hello` と打つと英語で返ってくることがあるので、日本語で話しかけるのがおすすめです。

  

```

> こんにちは

```

  

返事が返ってきたら、次にファイル作成の指示を出します。

  

```

> このフォルダに hello.txt を作って、中に「Hello, Claude Code」と書いて

```

  

ファイル作成の確認ダイアログが出るので、`y` または Enter で承認します。

  

> 📸 [SCREENSHOT-09] ファイル作成の承認ダイアログ

> *キャプション例：Claude Code は破壊的な操作の前に必ず確認を求めてくる*

  

`/exit` で REPL を抜けて、

  

```bash

cat hello.txt

```

  

で内容を確認してください。`Hello, Claude Code` と表示されれば成功です。自分の指示で AI がファイルを作った、その感覚を覚えておいてください。

## 詰まったときの対処

  

| 症状 | 対処 |

|---|---|

| `command not found: claude` | PATH が通っていない。Step 3 のコマンドを実行 |

| `claude doctor` で警告が出る | 表示されたメッセージに従う。多くは PATH または古いバージョンの指摘 |

| ブラウザログインが終わらない | ターミナルに表示された URL を手動コピーしてブラウザに貼り付け、ログイン後にコードをターミナルに戻す |

| `403 Forbidden` | claude.ai のプラン未加入。Pro / Max プランの加入状況を確認 |

| インストールスクリプトが HTML を返す | ネットワーク・地域制限の可能性。Homebrew (`brew install --cask claude-code`) で代替可能 |

  

困ったときの一手目は必ず `claude doctor` です。診断結果を読んでから対処してください。

  

> 📸 [SCREENSHOT-10] `claude doctor` の正常時の出力例

> *キャプション例：すべて緑のチェックが付いていれば問題なし*

## チェックリスト

  

次の項目がすべて確認できれば完了です。

  

- `claude --version` でバージョン番号が表示される

- `claude doctor` が問題なしを返す

- `git --version` が表示される

- `claude` で REPL に入り、「こんにちは」に返事が返ってきた

- `~/cc-practice/hello.txt` が作成されている

- `unset ANTHROPIC_API_KEY` で意図しない API キー利用を防いだ（該当者のみ）
